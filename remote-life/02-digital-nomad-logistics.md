# 数字游民实务：签证、税务、法律实体与医保

> **醒目免责声明**：本文仅供教育与参考，非法律/税务/移民/保险建议，规则因国籍与年份变化，且会受税收居民身份、居住地、签证类型与保险条款影响；行动前务必咨询专业人士，并核实最新官方信息。

> 远程生活的现实基建：签证选择、税务居民、法律实体、医保、跨境收款、时区协作与选址决策。

## 0. 本章定位：自由不是“飞走”，而是把约束显性化

本章服务《逃离坐班白皮书》的“退出与长期运行”阶段。

如果说 [安全退出过渡](01-safe-exit-transition.md) 解决的是“什么时候离开坐班”，本章解决的是“离开后如何不被现实基建绊倒”。

数字游民不是长期旅游，也不是逃避规则。

它更像把生活拆成七个可管理系统：身份、税务、法律主体、医保、金融、协作、选址。

学完后，你应能完成这些产出：

- 画出自己的“居住地—客户—公司—银行—保险”结构图。
- 用同一张表比较 3–5 个目的地的签证、税务、医保、网络与成本。
- 给税务师准备完整事实清单，而不是只问“我该交多少税”。
- 在个体户、有限公司/LLC、EOR、平台合同工之间做初步取舍。
- 选择适合远程工作的保险组合，并识别“看似便宜但不赔”的缺口。
- 计算团队跨时区 overlap hours，并据此设计 async 工作流。

## 2. 总体架构：数字游民的七层操作系统

| 层级 | 核心问题 | 典型风险 | 产出物 |
|---|---|---|---|
| 身份层 | 我能否合法停留？ | 逾期、入境被拒、签证条件违反 | 签证路线图 |
| 税务层 | 我在哪里是税收居民？ | 双重征税、漏报海外账户、社保义务 | 税务事实包 |
| 法律层 | 我用什么主体签合同？ | 无限责任、客户不信任、发票不合规 | 主体/合同结构 |
| 医疗层 | 生病或事故谁买单？ | 旅游险拒赔、既往症不赔、撤离无保障 | 保险矩阵 |
| 金融层 | 钱如何进来、换汇、留痕？ | 冻卡、高手续费、无法证明收入 | 收款与账务流程 |
| 协作层 | 跨时区如何交付？ | 永久熬夜、同步会议过多、响应预期混乱 | Async SOP |
| 选址层 | 去哪里住、住多久？ | 网络差、医疗弱、生活成本失控 | 目的地评分表 |

本章所有步骤都围绕这七层展开。

## 3. 签证：先区分“可停留”与“可工作”

### 3.1 三个概念不要混淆

第一，免签或旅游签通常允许你入境与旅游。

第二，工作许可通常允许你在当地为当地雇主或客户提供劳动。

第三，数字游民签证通常位于中间地带：允许你在该国居住，并为境外雇主或境外客户远程工作。

关键不是“我是否打开电脑”。

关键是当地法律如何定义工作、劳动市场、收入来源与税务连接。

如果你计划在一个地方连续住数月，并稳定远程工作，优先研究 digital nomad visa、remote work visa、temporary residence for remote workers。

### 3.2 数字游民签证的典型要求

不同国家差异很大，但常见材料包括：

- 有效护照，且有效期通常需覆盖停留期后 3–6 个月。
- 远程工作证明，例如雇佣合同、客户合同、自由职业证明、公司证明。
- 境外收入证明，例如银行流水、工资单、发票、税单。
- 最低收入门槛，可能按月收入、年收入或存款余额计算。
- 无犯罪记录证明，部分国家要求公证或认证。
- 医疗保险，通常要求覆盖整个停留期。
- 住宿证明，例如租约、酒店订单、地址声明。
- 申请费与可能的居留卡制卡费。
- 照片、申请表、动机信、行程计划。

建议为每个候选国家建立一张材料清单：

| 字段 | 你要填写的内容 |
|---|---|
| 官方项目名称 | 以政府官网名称为准 |
| 官方链接 | 只放政府、移民局、使馆链接 |
| 允许停留时长 | 首次批准、可续签次数、最长总时长 |
| 是否允许本地客户 | 允许 / 禁止 / 未明确 |
| 收入门槛 | 月收入、年收入、存款、家庭成员加成 |
| 保险要求 | 最低保额、是否必须本地保险 |
| 税务说明 | 是否有特殊税制、免税期、当地申报义务 |
| 家属政策 | 配偶、子女能否随行、是否可工作 |
| 申请地点 | 境外使领馆 / 境内在线 / 入境后转换 |
| 审理周期 | 官方周期与实际经验分开记录 |
| 风险备注 | 语言、材料认证、政策变动、配额 |

### 3.3 旅游签远程工作的灰区与风险

很多人会说：“我只是人在国外，客户在国外，又不抢当地工作。”

这句话在某些地方可能接近现实，但不能替代法律判断。

灰区常见风险包括：

- 入境官认为你不是短期旅游，而是在长期居住。
- 频繁入境导致被询问资金来源、住宿、离境计划。
- 在当地宣传接本地客户，被视为本地工作。
- 使用当地地址、手机号、银行、税号后形成居住事实。
- 租约、水电、会员、长期学校等证据显示你“实际居住”。
- 公司合规团队不允许员工在未授权国家工作。
- 保险公司以“非旅游目的”拒绝理赔。

如果仍计划使用旅游签短停，至少做三件事：

1. 阅读官方允许活动清单。
2. 控制停留天数与入境频率，避免形成事实居住。
3. 准备明确的离境机票、住宿、资金证明与解释口径。

不要声称你在当地工作。

不要伪造行程、合同或银行流水。

不要把“别人都这么做”当成合规依据。

### 3.4 Visa run：不是策略，只是风险缓冲

Visa run 指在签证或免签期限到期前离境，再重新入境以获得新的停留期。

每次入境都是一次新的边检判断。

风险信号包括：

- 短时间内多次出入境。
- 每次都停留接近上限。
- 没有稳定居住地或工作解释。
- 护照上出现明显“边境跳跃”模式。
- 回答不出为什么频繁返回同一国家。

更稳妥的做法：把 visa run 视为应急方案；喜欢某地就研究长期居留或数字游民签证；每年设定合规停留预算，不把最大天数用满。

### 3.5 代表性数字游民签证对比表

以下仅为学习用示例，政策、门槛、费用和税务待遇会变化。

申请前务必以官方移民局、政府门户、使领馆信息为准。

| 国家/地区 | 项目类型示例 | 常见停留框架 | 常见关注点 | 适合谁 |
|---|---|---:|---|---|
| 葡萄牙 | D8 / remote work residence | 通常可从临时居留走向续签 | 收入证明、住宿、税务居民、NIF、NHR 变化 | 想在欧洲长期布局的人 |
| 西班牙 | Digital Nomad Visa | 通常 1 年起，可续 | 收入门槛、社保/税务文件、客户来源比例 | 有稳定远程合同的人 |
| 爱沙尼亚 | Digital Nomad Visa | 通常最长约 1 年 | 收入门槛、停留天数、税务居民判断 | 偏技术、喜欢电子政务的人 |
| 克罗地亚 | Digital Nomad Residence | 通常最长约 1 年 | 住宿、保险、收入或存款证明 | 想低成本欧洲体验的人 |
| 希腊 | Digital Nomad Visa | 通常 1 年起，可转居留 | 税务、住宿、收入门槛、家庭成员加成 | 地中海生活偏好者 |
| 迪拜/UAE | Virtual Working Programme 等 | 通常约 1 年 | 收入、保险、居住成本、银行开户 | 高收入、重视税务与航线的人 |
| 格鲁吉亚 | 远程工作/长期停留相关安排 | 规则曾较灵活 | 税务居民、小企业税制、入境政策变化 | 预算敏感、喜欢高加索的人 |
| 墨西哥 | Temporary Resident 路线 | 通常 1 年起，可续 | 经济能力证明、领馆差异、税务连接 | 面向北美客户时区的人 |
| 哥斯达黎加 | Rentista / remote worker 类 | 通常 1 年起 | 收入、保险、文件认证 | 喜欢自然环境的人 |
| 巴巴多斯 | Welcome Stamp | 通常约 12 个月 | 收入门槛、成本、保险 | 高收入、短期海岛体验者 |

使用上表时，不要只比月收入门槛。

还要比较：是否允许带家属、是否能开本地银行账户、是否触发税务居民、是否要求本地租约、是否必须购买本地保险、是否需要公证认证、是否允许离境后保持居留、是否能转长期居留、是否与客户时区匹配。

### 3.6 签证选择的五步流程

#### 步骤 1：列出候选地

先从 8–10 个国家开始，不要一开始就追求最优。

来源包括客户时区、生活成本、语言能力、航线、医疗、安全、社群密度、签证可行性。

#### 步骤 2：淘汰硬性不满足项

硬性不满足项包括收入门槛达不到、护照不符合、必须有当地雇主、不能携带必要家属、保险要求无法满足、审理周期超过出发窗口。

#### 步骤 3：确认税务后果

一个签证允许你住，不代表税务简单。

要确认住满多少天会成为税收居民、是否按全球收入征税、是否有特殊税制、是否需要当地税号、是否有税收协定、是否有社会保险缴费义务。

#### 步骤 4：确认工作合规

逐条读 permitted activities。

确认是否只能为境外雇主工作、是否能服务本地客户、是否能开本地公司、是否能雇佣本地员工、是否能以自由职业者身份开票。

#### 步骤 5：做成本与风险决策

```text
首年总成本 = 申请费 + 文件认证 + 律师/代理 + 机票 + 押金 + 保险 + 当地生活成本 + 税务申报成本 + 应急金
```

完成标准：你能解释为什么选择 A 而不是 B，并能指出 A 的最大风险。

## 4. 税务：先问“我是谁的税收居民”

### 4.1 税务不是“我在哪个国家收钱”这么简单

远程收入常同时连接多个地方：你的人在哪里、客户在哪里、公司注册在哪里、银行账户在哪里、合同履行地在哪里、服务器与团队在哪里、家庭和资产在哪里。

税务机关通常不会只看一个因素，而会看事实组合。

所以不要把税务规划简化成“每个国家都不住满 183 天就不用交税”。

### 4.2 183 天规则只是入口，不是答案

很多国家有 183 天规则：在一个税务年度内居住达到一定天数，可能成为税收居民。

但常见判断还包括永久住所、主要生活利益中心、家属所在地、经济利益中心、当地工作许可、当地公司管理控制、规律返回某国、国内法与税收协定的 tie-breaker 规则。

正确表述是：183 天通常是重要阈值，但不是唯一规则；低于 183 天也可能有税务义务，高于 183 天也可能因协定或特殊规则产生不同结果。

### 4.3 税收居民 vs 来源地征税

| 概念 | 问题 | 常见后果 |
|---|---|---|
| 税收居民 | 你是否被某国视为“本国税务居民”？ | 可能对全球收入申报纳税 |
| 来源地征税 | 某笔收入是否来源于该国？ | 即使非居民，也可能被该国征税 |

举例：你住在国家 A，为国家 B 客户写代码，公司注册在国家 C，钱打到国家 D 的账户。

可能的问题包括：A 是否认为收入来源于 A；B 是否要求客户代扣预提税；C 是否对公司利润征税；D 是否要求账户信息申报或触发合规审查。

### 4.4 双重征税与税收协定

双重征税通常指同一笔收入被两个或多个税务管辖区征税。

缓解方式可能包括 Foreign Tax Credit、Exemption、Tax treaty、Tie-breaker。

但协定不是自动生效的护身符。

你可能需要申请税收居民证明、在申报表中披露协定适用、提供合同与付款记录、证明服务履行地点、遵守当地申报期限。

完成标准：你知道自己可能适用哪些协定，但不自行“脑补”结论。

### 4.5 中国籍远程工作者的高层次关注点

以下是教育性概览，不是税务建议。

中国税务居民规则通常会关注住所、居住天数、境内外所得等因素。

中国籍人士如果长期在海外居住，仍需认真判断：

- 是否仍为中国税收居民。
- 境外所得是否需要在中国申报。
- 是否有境内工资、劳务、股息、房租、投资收益。
- 是否有社保、公积金、个税汇算清缴问题。
- 是否通过境外公司收款但实际管理控制在中国。
- 是否需要处理外汇、银行流水与收入性质证明。

不要只问“我人在国外还要不要交中国税”。

应向税务师提供每年在中国停留天数、户籍、家庭、住房、社保状态、境内外收入明细、合同主体、收款账户、境外公司身份、股权激励信息。

### 4.6 美籍/绿卡持有人：全球申报是核心差异

美国公民和绿卡持有人通常需要按全球收入向美国申报。

常见关键词包括 FEIE、Foreign Tax Credit、FBAR、FATCA/Form 8938、Self-employment tax、State tax。

关键结论：美籍/绿卡数字游民通常不能用“我不住美国”来结束税务问题。

如果你是美国税务人，建议每年固定找熟悉 expat tax 的 CPA 或 EA。

### 4.7 带给会计师/税务师的决策清单

第一次咨询前，不要只发一句“我这种情况怎么省税”。

请准备以下材料：

- [ ] 护照国籍、绿卡/永居、长期居留身份。
- [ ] 过去 3 年每个国家的入境与离境日期。
- [ ] 未来 12 个月计划停留国家和日期。
- [ ] 当前雇主、客户、平台与合同副本。
- [ ] 每笔收入的来源国家、付款方、金额、币种。
- [ ] 收款账户清单：银行、Wise、Payoneer、Stripe、PayPal、加密资产账户。
- [ ] 公司或个体户注册文件。
- [ ] 股东、董事、实际控制人信息。
- [ ] 发票、收据、成本费用记录。
- [ ] 已缴税证明、预扣税凭证、社保缴费记录。
- [ ] 家庭、住房、保险、学校等生活中心信息。
- [ ] 期权、RSU、股息、利息、房租、资本利得明细。
- [ ] 希望解决的问题：合规、节税、移民路径、客户开票、公司结构。

可直接发送的问题模板：

```markdown
## 我要确认的问题
1. 2026 年我是否会成为 X 国税收居民？判断依据是什么？
2. 我从 Y 国客户获得的咨询收入，在 X 国和中国/美国分别如何处理？
3. 如果我注册 Z 地 LLC/有限公司，是否改变个人纳税义务？
4. 我的海外账户是否需要申报？门槛、表格、截止日是什么？
5. 我应如何保存合同、发票、成本与差旅记录？保存几年？
```

完成标准：会计师能在 30 分钟内理解事实，而不是花整场会议追问基础信息。

### 4.8 税务年度运行 SOP

建议每月做一次账务整理，不要等到报税季再回忆。

每月 SOP：

1. 导出所有收款账户流水。
2. 导出信用卡与银行卡支出。
3. 标记收入、成本、个人消费、转账。
4. 保存发票、合同、收据到同一云盘。
5. 记录当月各国停留天数。
6. 记录重大事件：搬家、签证获批、公司注册、客户变更。
7. 汇总当月外汇兑换日期、金额、汇率、手续费。

| 日期 | 国家 | 收入/支出 | 付款方/收款方 | 金额 | 币种 | 汇率 | 账户 | 类别 | 凭证链接 |
|---|---|---|---|---:|---|---:|---|---|---|

完成标准：任意一笔收入都能追溯到合同、发票、流水和交付记录。

## 5. 法律实体：主体是风险、税务与信任的接口

### 5.1 三种常见工作形态

远程变现通常有三类结构：个人/个体户直接接单；通过有限公司、LLC 或类似实体签合同；通过平台、EOR、agency 或 contracting marketplace 收款。

这些结构没有绝对优劣。

关键看收入规模、客户要求、责任风险、税务居民身份、长期居住地与合规成本。

可结合 [自由职业与远程合同](../income/02-freelancing-remote-contracting.md) 中的报价、合同与获客章节一起设计。

### 5.2 个体户/个人直接签约

适合场景：

- 年收入仍在验证阶段。
- 客户接受个人合同或平台合同。
- 项目责任较低。
- 你不需要雇佣团队。
- 发票与税务处理在居住地较简单。

优点：启动快、成本低、账务简单、不需要维护公司秘书、年审、复杂报表。

缺点：个人责任边界较弱、大客户可能不接受、税务优化空间有限、收入与个人账户混同风险高、某些国家要求注册自由职业者或缴社保。

### 5.3 有限公司/LLC/类似实体

公司结构可能有帮助，但不是魔法。

它常解决责任隔离、客户信任、发票合规、团队扩展、股权与品牌问题。

但它也带来注册、年审、会计、报税成本，公司所在地与个人所在地双重合规，CFC、实际管理地、常设机构、银行 KYC、分红和董事费等复杂问题。

如果你只是月收入 2,000 美元、客户 1–2 个、风险低，过早搭复杂离岸结构可能得不偿失。

### 5.4 EOR、平台与合同工

EOR（Employer of Record）常用于公司想雇你，但不想在你所在地设实体。

平台或 EOR 可能处理本地雇佣合同、工资发放、税务预扣、社保福利和合规文件。

适合你接近全职远程雇佣、客户公司有合规要求、你需要稳定工资单用于签证或租房、你不想维护公司实体。

缺点是费用较高、可选择国家有限、合同灵活性低、仍需判断个人税务居民、不一定支持频繁跨国移动。

### 5.5 主体选择对比表

| 结构 | 启动成本 | 维护成本 | 责任隔离 | 客户信任 | 税务复杂度 | 适合阶段 |
|---|---:|---:|---|---|---|---|
| 个人直接签约 | 低 | 低 | 弱 | 中低 | 中 | 验证服务、低风险项目 |
| 个体户/自由职业注册 | 中低 | 中低 | 弱至中 | 中 | 中 | 稳定接单、需开票 |
| 本地有限公司 | 中 | 中 | 中至强 | 高 | 中至高 | 本地长期居住、客户企业化 |
| 海外 LLC/公司 | 中至高 | 高 | 中至强 | 高 | 高 | 跨境客户、产品化、收入较高 |
| EOR | 低至中 | 中至高 | 由平台处理 | 高 | 中 | 远程雇佣、客户合规要求 |
| Marketplace 平台 | 低 | 平台抽成 | 中 | 中 | 中 | 早期获客、短项目 |

### 5.6 什么时候考虑注册公司

可以考虑注册公司的信号：

- 年收入稳定超过当地个人直接接单的舒适区。
- 客户要求公司发票、商业保险、MSA、DPA。
- 项目可能造成较大损失，例如安全、金融、医疗、生产系统。
- 你开始雇佣承包商或转包工作。
- 你计划做 SaaS、插件、课程、咨询品牌。
- 你需要更清晰的成本归集与财务报表。
- 你希望未来出售资产、引入合伙人或签长期合同。

暂缓注册公司的信号：没有稳定收入、只是为了“看起来专业”、不了解公司所在地报税义务、无法承担会计合规费用、准备频繁换税务居民地、希望用公司隐藏个人收入来源。

### 5.7 合同条款最小清单

至少检查工作范围、付款、税费、知识产权、保密、责任限制、终止、适用法律、数据保护、工作地点。

工程咨询合同里建议把“跨境远程工作”和“生产系统访问限制”列入谈判清单。

示例：

```text
Contractor may perform services remotely from locations outside Client's country,
provided that Contractor remains responsible for complying with applicable visa,
tax, and local laws. Client approval is required before accessing production
systems from a sanctioned or restricted jurisdiction.
```

这不是通用法律建议，只是提醒你把跨境远程工作写进谈判清单。

## 6. 医保：旅游险不是万能医保

### 6.1 先区分三类保障

| 类型 | 主要用途 | 常见缺口 |
|---|---|---|
| Travel medical insurance | 旅行期间突发疾病、意外、紧急撤离 | 既往症、长期慢病、常规体检、居住国保障有限 |
| Comprehensive international health insurance | 长期国际医疗、住院、门诊、慢病、可续保 | 保费高、等待期、区域限制、核保复杂 |
| Local public/private health plan | 某一国家本地医疗系统 | 跨国失效、语言与资格限制、等待期 |

短期试运行可以用 travel medical。

长期数字游民更应评估 comprehensive international health insurance。

如果你有家属、慢病、运动风险或计划长期海外，不能只买最便宜的旅游险。

### 6.2 选择保险时逐项检查

至少看这些条款：

- 保障地区：是否包括美国、加拿大、香港、新加坡等高成本地区。
- 单次旅行最长天数：90 天、180 天还是无限。
- 是否要求你从本国出发。
- 是否覆盖你长期居住的国家。
- 急诊、住院、门诊、处方药的保额。
- 既往症是否排除。
- 癌症、心理健康、康复、牙科、眼科是否覆盖。
- 运动风险：潜水、摩托、滑雪、攀岩是否排除。
- 医疗转运与遗体运返。
- 免赔额、共付比例、年度上限。
- 直付网络：是否可 direct billing。
- 续保保证：是否 guaranteed renewable。
- 理赔材料语言与时限。

### 6.3 常见不赔场景

| 场景 | 为什么容易不赔 | 预防动作 |
|---|---|---|
| 骑摩托摔伤 | 未持当地有效驾照或未戴头盔 | 查条款、持照、戴头盔、拍照留证 |
| 慢病复诊 | 被视为既往症或常规护理 | 购买覆盖既往症的长期险 |
| 长住某国 | 旅游险只覆盖旅行，不覆盖居住 | 选长期国际医保或本地医保 |
| 高风险运动 | 条款明确排除 | 购买运动附加险 |
| 战乱/制裁地区 | 地区排除 | 避开或购买专门保障 |
| 酒后事故 | 条款排除酒精/药物影响 | 不酒后驾驶和冒险活动 |
| 未及时通知保险公司 | 违反理赔流程 | 入院前或尽快联系 24h hotline |

### 6.4 医疗应急包

出发前准备：护照扫描件、保险保单 PDF、保险 24 小时救援电话、既往病史英文摘要、常用药通用名、过敏史、紧急联系人、血型、电子处方、当地急救电话、最近三家可英语沟通的医院地址。

建议把这些信息放在手机离线笔记和纸质卡片中。

完成标准：你昏迷时，同行者也能在 3 分钟内找到保险与病史信息。

## 7. 收款与银行：钱进得来、说得清、换得少

### 7.1 跨境收款的核心目标

好的收款系统满足四个条件：客户愿意付款；费用与汇率可控；银行不会因 KYC 不清楚频繁冻结；税务与会计能解释每笔钱的性质。

常见工具包括 Wise Business / Wise Personal、Payoneer、Stripe、PayPal、Deel、Remote、Oyster、本地银行账户、公司银行账户或 fintech business account。

工具可用性取决于你的国籍、居住地、公司注册地和行业。

### 7.2 Wise/Payoneer 等多币种账户的使用逻辑

多币种账户的价值是减少中间行费用、提高客户付款便利性。

但它们不是万能银行。

你要确认是否支持你的个人/公司注册地，是否提供 USD ACH、EUR IBAN、GBP sort code，是否可出具账户证明，是否支持客户所在国家付款方式，是否允许你的业务类型，是否能导出 CSV，汇率加价与固定手续费，大额入账 KYC 文件要求，资金是否有存款保护。

### 7.3 FX 成本计算

不要只看转账手续费。

```text
总成本 = 固定手续费 + 百分比手续费 + 汇率点差 + 中间行费用 + 收款行费用
```

举例：

```text
客户支付：10,000 USD
平台手续费：1.0% = 100 USD
固定手续费：10 USD
实际兑换汇率：1 USD = 7.10 CNY
中间市场汇率：1 USD = 7.18 CNY
汇率点差损失：(7.18 - 7.10) * 9,890 = 791.2 CNY
```

如果你每月收 10,000 USD，1% 的额外成本一年就是 1,200 USD。

完成标准：每条收款路径都算出真实到账金额，而不是只看“手续费低”。

### 7.4 收款路径设计

建议至少有两条可用路径：主路径负责默认付款，备份路径用于主账户风控或银行维护。

```text
US 客户 → ACH → Wise USD account → 保留 USD 支付工具/保险/机票
EU 客户 → SEPA → Wise EUR account → 部分兑换为当地生活费
平台客户 → Payoneer → 公司账户 → 会计入账
高信任长期客户 → Bank wire → 公司银行 → 月度对账
```

### 7.5 银行 KYC 文件包

提前准备护照、地址证明、居留许可或签证、公司注册文件、股东/董事名册、客户合同、发票样本、网站、LinkedIn、GitHub、作品集、最近 3–6 个月银行流水、税号或税务居民证明、业务说明。

业务说明模板：

```text
I provide remote software engineering consulting services to business clients.
Typical services include backend architecture, LLM application integration,
cloud infrastructure review, and technical due diligence. Clients are located
primarily in [countries]. Average invoice size is [amount]. Payments are made
under written service agreements and invoices.
```

完成标准：账户要求补充资料时，你能当天提交完整包。

## 8. 时区管理：不要用意志力对抗地球自转

### 8.1 选择基地前先算 overlap hours

时区管理不是“我愿意熬夜”。

长期熬夜会吞掉你从坐班逃出来的自由。

核心指标是 overlap hours：你和客户/团队都处在可工作时间的交集。

```text
overlap = intersection(你的工作窗口, 对方工作窗口)
```

假设你在里斯本，工作窗口是 09:00–17:00；纽约客户工作窗口是 09:00–17:00 Eastern Time。

夏令时里斯本约 UTC+1，纽约约 UTC-4。

纽约 09:00–17:00 对应里斯本 14:00–22:00。

你若坚持 09:00–17:00，只 overlap 14:00–17:00，约 3 小时。

如果把工作窗口调为 11:00–19:00，overlap 变为 14:00–19:00，约 5 小时。

### 8.2 Worked example：清迈服务旧金山客户

假设：

- 你在清迈：UTC+7。
- 客户在旧金山：冬季 UTC-8，夏季 UTC-7。
- 客户工作时间：09:00–17:00 Pacific Time。
- 你希望工作时间：08:00–18:00 Chiang Mai。

冬季换算：

```text
旧金山 09:00 = 清迈次日 00:00
旧金山 17:00 = 清迈次日 08:00
```

交集：你的 08:00–18:00 与客户 00:00–08:00 几乎没有交集。

如果你每天 07:30 开始，只能赶上客户 16:30–17:00，约 0.5 小时。

解决方案：

1. 把清迈作为 deep work base，只接 async-heavy 项目。
2. 约定每周 2 次固定 overlap call，例如清迈 08:00 / 旧金山前日 17:00。
3. 所有需求用 GitHub Issues、Linear、Notion 或 Jira 写清楚。
4. 每天在客户下班前提交 written update。
5. 对紧急响应另收 on-call premium，且限定次数。

### 8.3 Async 工作流模板

```markdown
## Daily async update — YYYY-MM-DD

### Done
- 完成了什么，链接 PR / commit / doc。

### Next
- 下一步做什么，预计完成时间。

### Blockers
- 需要谁提供什么信息，截止时间。

### Decisions needed
- 选项 A：影响、成本、风险。
- 选项 B：影响、成本、风险。
- 我的建议：选择 A，因为……
```

会议压缩原则：状态同步改文字；决策会议提前发选项；15 分钟能解决的不开 60 分钟；每周固定 1–2 个 overlap block；跨时区会议默认录制或写纪要。

可与 [独处生产力系统](03-solo-productivity-system.md) 配合，建立深度工作块、异步更新与复盘节奏。

## 9. 选址：用评分表，而不是用滤镜选生活

### 9.1 候选基地的六个硬指标

选基地时优先看签证可行性、税务后果、与客户时区 overlap、互联网稳定性、医疗可达性、总生活成本。

风景、咖啡馆、海滩、社群都重要，但不能排在硬指标前面。

### 9.2 评分 Rubric

给每个城市按 1–5 分评分。

| 维度 | 权重 | 1 分 | 3 分 | 5 分 |
|---|---:|---|---|---|
| 签证 | 20% | 只能短停且不确定 | 可停 90–180 天 | 有明确长期/游民路径 |
| 税务 | 20% | 高度不确定 | 可咨询后管理 | 规则清晰、合规成本可控 |
| 时区 | 15% | overlap < 1h | overlap 2–4h | overlap > 5h |
| 网络 | 15% | 不稳定 | 够用但需备份 | 光纤/5G/ coworking 稳定 |
| 医疗 | 10% | 英语医疗弱 | 基础可用 | 国际医院可达 |
| 成本 | 10% | 超预算 30%+ | 接近预算 | 低于预算且质量好 |
| 社群 | 5% | 孤立 | 有少量活动 | 有高质量专业社群 |
| 安全 | 5% | 风险高 | 可管理 | 安全感强 |

```text
总分 = Σ(维度分数 * 权重)
```

| 城市 | 签证 | 税务 | 时区 | 网络 | 医疗 | 成本 | 社群 | 安全 | 加权总分 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|
| 里斯本 | 4 | 3 | 4 | 4 | 4 | 2 | 5 | 4 | 3.65 |
| 清迈 | 3 | 3 | 1 | 4 | 3 | 5 | 4 | 4 | 3.20 |
| 墨西哥城 | 4 | 3 | 5 | 4 | 4 | 3 | 4 | 3 | 3.90 |

这些分数只是演示，你要根据自己的国籍、客户、收入、语言和风险偏好重算。

可参考：

- Nomad List：https://nomadlist.com/
- 官方移民局与使馆网站。
- Numbeo：https://www.numbeo.com/cost-of-living/
- Speedtest Global Index：https://www.speedtest.net/global-index
- SafetyWing、Cigna、Allianz 等保险条款页。
- 本地 expat forum、Reddit、Facebook group，但只能作为经验参考。
- Google Maps 搜索医院、coworking、健身房、超市。

使用社区信息时要标注日期。

数字游民政策变化很快，2023 年的帖子可能已经失效。

### 9.4 试住策略

不要第一次就签 12 个月租约。

建议第一站住 2–4 周，选择可取消住宿，前 3 天测试网络、噪音、通勤、健身、饮食，第 1 周拜访 coworking 与医院，第 2 周参加本地社群活动，第 3 周评估是否延长。

完成标准：你至少经历过一个完整工作周，再决定是否长期留下。

- [ ] eSIM 或本地 SIM 方案。
- [ ] 噪音耳机。
- [ ] 备用网络：手机热点、第二运营商、coworking。
- [ ] 和客户确认时区、响应时间、会议窗口。

### 10.7 生活与应急

- [ ] 家人或紧急联系人知道你的行程。
- [ ] 购买可改签或可取消的关键机票/住宿。
- [ ] 准备至少 3 个月生活费应急金。
- [ ] 了解目的地报警、急救、使领馆联系方式。
- [ ] 设置手机丢失应急流程。
- [ ] 保留国内地址、手机号、银行验证方式。

## 11. 30/60/90 Setup Plan

### 出发前 30 天：降低不可逆风险

第 1 周：建立 `nomad-operating-file`，写完个人事实页，导出过去 12 个月收入支出，列出 3 个候选目的地，用官方链接确认入境与签证要求。

第 2 周：预约税务师；如涉及长期居留，预约移民律师或持牌顾问；如准备注册公司，预约公司秘书、律师或会计；向保险经纪或保险公司确认条款。

第 3 周：建立收款主路径和备份路径，整理 KYC 文件包，购买保险，设置记账表，设置行程天数追踪。

第 4 周：和客户确认时区与 async SOP，把会议集中到 overlap block，完成设备备份，预订前 2–4 周住宿，设置家人应急联络机制。

### 出发后 60 天：验证基地是否适合工作

第 1–2 周：测试住处网络，记录上下行与延迟，找到最近医院、药店、coworking，办理 SIM/eSIM，确认银行卡取现与刷卡可用。

第 3–4 周：统计实际 overlap hours，记录每次会议是否影响睡眠，测试 async update 是否减少会议，评估客户满意度和交付速度。

第 5–8 周：计算真实月成本，评估签证延长或下一站方案，咨询是否触发当地税务居民或登记义务，评估医疗、社群、安全、运动、饮食。

完成标准：你能用数据回答“这里适不适合作为 3–6 个月基地”。

### 出发后 90 天：做结构化复盘

复盘问题：实际生活成本比预算高/低多少，哪项成本最意外，工作效率提高还是下降，客户是否感受到时区风险，签证路线是否可持续，税务事实是否变复杂，保险是否覆盖真实风险，是否需要更换收款工具，是否需要注册主体或改用 EOR，下一站是否应更靠近客户时区。

| 结果 | 行动 |
|---|---|
| 基地适合 | 延长住宿，完善签证和税务结构 |
| 工作适合但生活不适合 | 保留时区，换城市 |
| 生活适合但工作不适合 | 改客户结构或换基地 |
| 签证/税务风险过高 | 退出该地，重新选址 |
| 成本失控 | 缩短停留，回到低成本基地 |

## 12. 完整实操示例：远程工程顾问选择第一站

### 12.1 背景假设

假设你是中国籍资深后端/AI 工程顾问。

你的情况：客户主要在美国西海岸和欧洲；月收入 12,000 USD；通过个人合同收款，准备半年内考虑公司结构；希望每年有 6 个月在海外；不持有美国绿卡或美国公民身份；无家属随行；需要每周 2 次客户会议。

### 12.2 候选地

你列出：墨西哥城靠近美国时区；里斯本欧洲社群强且通往欧盟方便；清迈成本低且亚洲生活舒适。

### 12.3 签证初筛

| 城市 | 签证初筛 | 结论 |
|---|---|---|
| 墨西哥城 | 研究 temporary resident 或短停规则 | 适合北美客户，需确认领馆经济能力要求 |
| 里斯本 | 研究葡萄牙 D8 | 适合长期欧洲路线，材料较多 |
| 清迈 | 研究泰国长期签证/短停规则 | 成本低，但远程工作与长期停留需谨慎确认 |

### 12.4 时区计算

美国西海岸客户：墨西哥城 overlap 高，里斯本中等，清迈很低。

欧洲客户：里斯本 overlap 高，墨西哥城中低，清迈中等偏低。

如果收入 70% 来自美国西海岸，墨西哥城优先级上升；如果收入 70% 来自欧洲，里斯本优先级上升。

### 12.5 税务咨询准备

你给税务师的问题：

1. 如果我一年在中国 150 天、墨西哥 120 天、葡萄牙 60 天、其他国家 35 天，我的中国税收居民判断如何？
2. 墨西哥短住是否会触发当地税务登记或申报？
3. 如果未来用香港/新加坡/美国 LLC 或本地公司收款，个人纳税义务如何变化？
4. 我是否应先以个人接单，等收入稳定后再注册主体？
5. 需要保存哪些跨境服务证据？

### 12.6 决策

如果你最重视客户时区和收入稳定，先去墨西哥城试住 30 天。

如果你最重视长期欧洲居住路径，准备葡萄牙或西班牙材料。

如果你最重视低成本和深度工作，清迈可作为 async 项目基地，但不要承诺美国客户每日同步会议。

完成标准：你不是因为“哪里火”而出发，而是因为该地符合当前收入结构。

## 13. 常见坑与排查表

| 症状 | 可能原因 | 排查动作 |
|---|---|---|
| 入境时被反复询问 | 停留模式像长期居住 | 准备离境机票、住宿、资金证明，减少连续 visa run |
| 客户突然不让你从某国访问系统 | 合规、制裁、数据保护限制 | 提前提交工作地点清单，使用客户批准的访问方案 |
| 收款账户被冻结 | KYC 不完整或交易模式异常 | 提交合同、发票、网站、业务说明、税务文件 |
| 报税季崩溃 | 没有月度记账 | 每月导出流水，按收入/成本/转账分类 |
| 保险拒赔 | 条款排除长期居住、既往症或运动 | 购买前逐条确认 exclusions |
| 每天凌晨开会 | 基地与客户时区不匹配 | 改 async SOP，设 overlap block，换基地或换客户结构 |
| 公司注册后更复杂 | 没考虑个人税务和管理地 | 找跨境税务师审查结构，必要时简化 |
| 生活成本超预算 | 只看短视频价格 | 用真实租金、押金、保险、交通、外食重算 |
| 租房网络很差 | 入住前未测速 | 要求房东提供 Speedtest，首周住可取消住宿 |
| 目的地社群很热闹但无法工作 | 社交选择压过工作系统 | 先保证工作块、网络、睡眠，再参加活动 |

## 16. 最后提醒：合规是自由的基础设施

数字游民的目标不是把规则绕过去。

目标是把规则变成可管理的边界。

当签证、税务、主体、医保、收款、时区和选址都被系统化，你才真正拥有地点自由。

如果这些基础设施没有搭好，所谓自由会变成焦虑：怕入境、怕报税、怕生病、怕账户冻结、怕客户找不到你。

本章的最小行动不是立刻买机票。

而是今天就建立你的 `nomad-operating-file`，填完个人事实页，并选择 3 个候选地做第一轮评分。

完成它，再出发。
