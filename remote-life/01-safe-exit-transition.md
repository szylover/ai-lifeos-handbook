# 安全离职：跑道、F-you money 与过渡风控

> 用规则替代冲动：算清跑道、设定离职触发条件、按清单安全退出。

本文仅供教育与参考，非专业法律、税务、投资或医疗建议；涉及合同、竞业、税务、保险、贷款、投资配置时，请结合自身情况咨询持牌专业人士。

## 0. 本章定位：退出不是情绪动作，而是系统迁移

《逃离坐班白皮书》的主线是：用技能、市场、变现、资本四种杠杆，逐步换回时间、地点、财务与精神自由。

本章处在“退出”阶段，但它不是鼓励你立刻辞职。

更准确地说，本章教你把“我受够了”翻译成一组可验证的退出条件。

如果你还没有读总纲，先看 [逃离坐班白皮书总纲](../00-manifesto.md)。

如果你还没有理解 FIRE 的基本数学，先看 [FIRE 基础](../investing/01-fire-basics.md)。

如果你还没有搭建副业收入地图，配合阅读 [变现地图](../income/01-monetization-landscape.md)。

如果你缺少离职后的单人执行系统，后续接上 [Solo Productivity System](03-solo-productivity-system.md)。

本章的目标不是“勇敢裸辞”。

本章的目标是“让离职成为一项可回滚、可监控、可复盘的工程变更”。

### 学完本章你能交付什么

1. 你能用公式算出自己的 runway，并区分 emergency fund、runway 与 F-you money。
2. 你能给“什么时候辞职”写出明确触发条件，而不是靠情绪判断。
3. 你能设计“在职 → 副业 → 兼职/顾问 → 独立”的分阶段迁移路线。
4. 你能在离职前完成财务、保险、合同、客户、知识交接的检查清单。
5. 你能写出一段体面、可谈判、保留桥梁的离职沟通脚本。
6. 你能为副业断流、健康问题、家庭变化、市场下行准备 contingency plan。
7. 你能做出 6–12 个月的月度转型计划，并用指标判断是否继续、暂停或回撤。

### 本章前置假设

- 你是一名资深软件工程师，当前有主业工资收入。
- 你想减少对坐班雇佣关系的依赖，但不想用破产风险换自由幻觉。
- 你有能力做自由职业、咨询、产品、内容、远程合同工或 AI 自动化服务中的至少一种尝试。
- 你愿意用表格、日历、现金流记录和 pipeline 数据做决策。
- 你接受一个事实：安全离职通常不是一天完成，而是 6–24 个月的迁移项目。

## 1. 为什么冲动裸辞经常失败

冲动裸辞通常不是因为人不努力失败。

它失败在系统边界条件没有建好。

### 1.1 冲动裸辞的典型触发器

- 周会里被公开否定。
- 连续加班后身体报警。
- 晋升失败或调薪低于预期。
- 新领导上任，价值观冲突。
- 看到别人远程旅行、独立开发、自由职业的高光片段。
- 副业第一个月赚了钱，于是把一次性收入误判为稳定收入。
- 投资账户上涨，于是把市场 beta 误判为个人现金流能力。

这些触发器都真实。

但真实不等于足以支持退出。

### 1.2 冲动裸辞的失败链路

1. 情绪峰值：当下痛苦超过理性阈值。
2. 决策收缩：只看“离开”的好处，不看“离开后每天如何产生现金流”。
3. 现金流真空：工资停止，副业还没有稳定获客。
4. 固定成本刚性：房租、房贷、家庭支出、保险、赡养责任不会因为你自由而消失。
5. 心理压力放大：每一笔支出都变成倒计时。
6. 定价下坠：为了快点成交，接低价、坏客户、模糊需求。
7. 交付质量下降：压力过大导致效率与判断力下降。
8. 自我怀疑：把“准备不足”误读成“我不适合自由职业”。
9. 被迫回职场：仓促找工作，谈判力比离职前更弱。

### 1.3 用规则替代冲动

安全离职的核心不是压抑情绪。

核心是把情绪作为信号，把行动交给规则。

你可以这样定义规则：

- 情绪规则：连续 4 周出现强烈离职冲动，进入“退出评估流程”，但不当天辞职。
- 财务规则：runway 未达到目标前，不裸辞。
- 收入规则：副业收入未满足连续性条件前，不全职独立。
- 法务规则：未读完雇佣合同、竞业、IP、moonlighting 条款前，不签外部客户合同。
- 健康规则：如果身体已经报警，优先请假、医疗评估、降载，而不是直接把所有风险转移到无收入状态。
- 回撤规则：如果离职后指标连续 2–3 个月低于红线，执行 re-entry plan。

### 1.4 把离职当作生产变更

工程师熟悉生产变更。

上线前你会做容量评估、回滚方案、监控、告警、灰度。

离职也一样。

| 生产变更 | 安全离职 |
|---|---|
| 容量评估 | runway、固定成本、收入波动 |
| 灰度发布 | 在职副业、兼职顾问、减少工时 |
| 监控指标 | 月净支出、收入连续性、pipeline、健康指标 |
| 告警阈值 | 现金 runway 低于 9 个月、pipeline 低于 2 个月 |
| 回滚方案 | re-entry plan、合同工、前雇主返聘、求职 pipeline |
| 事后复盘 | 每月财务与客户复盘 |

离职不是“删库”。

离职应该是“带备份、带监控、可回滚的架构迁移”。

## 2. 三个现金概念：emergency fund、runway、F-you money

涉及现金、投资、贷款和保险的判断都属于个人财务决策；本文仅供教育与参考，非投资建议。

### 2.1 Emergency fund：应急金

Emergency fund 是为了处理短期突发事件的现金缓冲。

它对应的问题是：“如果明天车坏了、家人生病、客户延期付款，我会不会被迫卖资产或借高息债？”

关于应急金和现金流桶的完整方法，见 [应急金与现金流](../investing/04-emergency-fund-cashflow.md)。

常见目标：

- 单身、固定成本低、技能强：3–6 个月必要支出。
- 有房贷、家庭责任、收入波动：6–12 个月必要支出。
- 准备离职并转独立：应急金只是底座，不能替代 runway。

应急金放置原则：

- 高流动性。
- 低波动。
- 可在 1–3 个工作日内使用。
- 不追求收益最大化。
- 不放在需要承担大幅波动的资产里。

### 2.2 Runway：离职后还能飞多久

Runway 是你在没有主业工资的情况下，靠可动用流动资产覆盖净支出的月份数。

公式：

$$
\text{Runway（月）}=\frac{\text{可动用流动资产}}{\text{月净支出}}
$$

其中：

- 可动用流动资产 = 现金 + 货币基金 + 短期定存 + 可快速变现且你愿意卖出的低波动资产。
- 月净支出 = 月总支出 - 离职后稳定现金流。
- 不建议把自住房、退休账户、长期锁定资产、未归属 RSU、未到账奖金算作可动用流动资产。
- 如果你的副业收入不稳定，只能按保守折扣计入稳定现金流。

更保守的公式：

$$
\text{保守 Runway}=\frac{\text{现金类资产}-\text{一次性离职成本}-\text{预留应急金}}{\text{必要月支出}-\text{保守稳定收入}}
$$

目标区间：

- 6 个月：只适合已有强 pipeline、低固定成本、可快速回职场的人。
- 12 个月：多数单身资深工程师的最低安全线。
- 18 个月：适合需要建设产品、内容、咨询口碑的人。
- 24 个月：适合有家庭责任、房贷、签证限制、健康风险或市场下行担忧的人。

### 2.3 F-you money：拒绝坏交易的选择权

F-you money 不是“永远不用工作的钱”。

那更接近 FIRE，见 [FIRE 基础](../investing/01-fire-basics.md)。

F-you money 是你可以拒绝不合理雇主、坏客户、低价合同、损害健康安排的现金与资产缓冲。

它解决的问题是：“我能不能说不？”

一个实用定义：

$$
\text{F-you money}=\text{足以支持你拒绝坏交易并寻找下一种选择的资产缓冲}
$$

可操作版本：

- 初级 F-you money：6 个月必要支出，能拒绝短期恶劣项目。
- 中级 F-you money：12–18 个月必要支出，能从容转岗、转客户、转赛道。
- 高级 F-you money：24+ 个月必要支出，能做产品、写作、咨询品牌等慢变量。

### 2.4 三者的区别

| 概念 | 解决的问题 | 常见规模 | 是否用于日常消耗 | 关键约束 |
|---|---|---:|---|---|
| Emergency fund | 突发事故 | 3–12 个月必要支出 | 只在紧急时用 | 高流动、低波动 |
| Runway | 离职后活多久 | 12–24 个月目标 | 可以按计划消耗 | 按月监控 burn rate |
| F-you money | 能否拒绝坏交易 | 6–24+ 个月 | 不鼓励轻易消耗 | 提供谈判权与心理安全 |
| FIRE 资产 | 是否可长期不工作 | 25–33 倍年支出常见 | 按提取率使用 | 依赖长期资产配置 |

## 3. 从零算一遍：runway 工作示例

假设你是一名资深后端工程师，准备从坐班转向远程咨询与 AI 自动化服务。

### 3.1 当前资产与支出

| 项目 | 金额 | 是否计入可动用流动资产 | 说明 |
|---|---:|---|---|
| 活期现金 | ¥80,000 | 是 | 可立即使用 |
| 货币基金 | ¥120,000 | 是 | 低波动，T+1 |
| 3 个月定存 | ¥100,000 | 是 | 可提前支取但可能损失利息 |
| 股票基金 | ¥180,000 | 部分 | 波动较大，保守只计 50% |
| 未归属 RSU | ¥200,000 | 否 | 没归属就不是你的现金 |
| 自住房净值 | ¥800,000 | 否 | 不能轻易用于生活费 |
| 年终奖预期 | ¥100,000 | 否 | 未到账不计入 |

保守可动用流动资产：

$$
\text{可动用流动资产} = 80{,}000 + 120{,}000 + 100{,}000 + 180{,}000 \times 0.5 = 390{,}000
$$

### 3.2 当前月支出

| 支出项 | 金额/月 | 可削减性 | 离职后目标 |
|---|---:|---|---:|
| 房租/房贷 | ¥9,000 | 低 | ¥9,000 |
| 餐饮 | ¥4,500 | 中 | ¥3,800 |
| 交通 | ¥1,500 | 高 | ¥800 |
| 保险 | ¥1,200 | 低 | ¥1,500 |
| 家庭支持 | ¥3,000 | 低 | ¥3,000 |
| 订阅与工具 | ¥1,200 | 中 | ¥800 |
| 学习/社交 | ¥2,000 | 中 | ¥1,000 |
| 旅行娱乐 | ¥3,000 | 高 | ¥1,500 |
| 税务/会计预留 | ¥0 | 必须新增 | ¥1,500 |
| 合计 | ¥25,400 |  | ¥22,900 |

离职后必要月支出按 ¥23,000 估算。

### 3.3 副业收入折扣

最近 6 个月副业收入：

| 月份 | 收入 | 来源 |
|---|---:|---|
| M-6 | ¥0 | 无 |
| M-5 | ¥8,000 | 一次性脚本项目 |
| M-4 | ¥12,000 | 顾问小时包 |
| M-3 | ¥18,000 | AI 自动化项目 |
| M-2 | ¥15,000 | 维护合同 + 小项目 |
| M-1 | ¥22,000 | 顾问 + 维护 |

不要直接把最近一个月 ¥22,000 当稳定收入。

保守估计可以用三个数：

- 最近 6 个月中位数：¥13,500。
- 最近 3 个月平均值：¥18,333。
- 已签约未来 3 个月 MRR：¥8,000。

安全计算时只计入已签约或高确定性的稳定部分，例如 ¥8,000。

于是：

$$
\text{月净支出}=23,000-8,000=15,000
$$

$$
\text{Runway}=\frac{390,000}{15,000}=26\text{个月}
$$

如果不计副业收入：

$$
\text{Runway}=\frac{390,000}{23,000}\approx16.96\text{个月}
$$

### 3.4 结论不要只看一个数字

| 口径 | Runway | 解释 | 决策含义 |
|---|---:|---|---|
| 乐观口径：计最近月副业 ¥22,000 | 390 个月 | 月净支出只剩 ¥1,000 | 明显不可信 |
| 中性口径：计 6 个月中位数 ¥13,500 | 41 个月 | 有一定参考 | 仍需验证连续性 |
| 保守口径：只计已签约 ¥8,000 | 26 个月 | 可用于决策 | 进入可评估区间 |
| 压力口径：不计副业 | 17 个月 | 最坏现金流 | 若可接受，安全性强 |

安全离职的建议：

- 决策时看保守口径。
- 压力测试看不计副业口径。
- 乐观口径只用于 upside 规划，不用于辞职触发。

## 4. 建一个可复制的 Runway 表

你可以用 Excel、Google Sheets、Notion Database、Airtable 或纯 CSV。

最小字段如下。

### 4.1 资产表

| 字段 | 示例 | 说明 |
|---|---|---|
| account | 招行活期 | 账户名 |
| category | cash | cash / money_market / deposit / taxable_investment |
| balance | 80000 | 当前余额 |
| liquidity_days | 0 | 几天内可用 |
| haircut | 1.0 | 保守折扣，股票基金可用 0.5 |
| usable_amount | 80000 | balance × haircut |
| include_in_runway | TRUE | 是否计入 runway |

### 4.2 支出表

| 字段 | 示例 | 说明 |
|---|---|---|
| category | rent | 支出类别 |
| monthly_amount | 9000 | 月金额 |
| required | TRUE | 是否必要 |
| reducible | LOW | HIGH / MID / LOW |
| target_after_quit | 9000 | 离职后目标 |
| cut_action | 不续租时换小房 | 降本动作 |

### 4.3 收入表

| 字段 | 示例 | 说明 |
|---|---|---|
| source | Client A maintenance | 收入来源 |
| type | retainer | retainer / project / product / content / salary |
| monthly_amount | 8000 | 月度金额 |
| signed | TRUE | 是否已签约 |
| months_committed | 3 | 承诺月数 |
| collection_risk | LOW | 收款风险 |
| include_as_stable | TRUE | 是否计入稳定收入 |

### 4.4 指标表

| 指标 | 公式 | 目标 |
|---|---|---|
| usable_liquid_assets | SUM(usable_amount where include) | 持续增长 |
| necessary_monthly_spend | SUM(target_after_quit where required) | 可解释 |
| stable_side_income | SUM(income where signed and include_as_stable) | 连续增长 |
| net_burn | necessary_monthly_spend - stable_side_income | 逐月下降 |
| conservative_runway | usable_liquid_assets / net_burn | 12–24+ |
| no-income_runway | usable_liquid_assets / necessary_monthly_spend | 12+ 更稳 |

### 4.5 每月更新流程

1. 每月最后一天导出银行、信用卡、券商账户余额。
2. 更新资产表。
3. 从账单软件或银行卡流水汇总支出。
4. 标记哪些支出是必要支出，哪些是 lifestyle inflation。
5. 更新副业收入的实收金额，而不是开票金额。
6. 单独记录应收账款与逾期账款。
7. 计算保守 runway 与压力 runway。
8. 如果 runway 低于目标，暂停扩大支出。
9. 如果 runway 连续 3 个月增长，同时副业 pipeline 增长，才进入下一阶段。

完成标准：你能在 15 分钟内说清“如果今天离职，我最坏能撑几个月”。

## 5. 副业收入爬坡：从工资依赖到多现金流

离职前最重要的不是“找到热爱的事”。

最重要的是证明你能在没有雇主分配任务的情况下持续获得市场付款。

### 5.1 四阶段迁移路径

| 阶段 | 身份 | 收入结构 | 主要任务 | 退出到下一阶段条件 |
|---|---|---|---|---|
| A | 全职雇员 | 工资 90%+ | 降本、验证技能、建立获客面 | 每周稳定 5–10 小时副业投入 |
| B | 全职雇员 + 副业 | 工资 + 项目 | 形成案例、报价、交付 SOP | 副业连续 6 个月覆盖生活成本 X% |
| C | 兼职/顾问过渡 | 工资/顾问 + 副业 | 减少主业工时，放大高价服务 | pipeline 覆盖未来 3–6 个月 |
| D | 全职独立 | 多客户/产品/内容 | 交付、获客、系统化 | 不再依赖单一客户或平台 |

### 5.2 触发条件的 X% 怎么定

常见规则：

- 保守型：副业月收入连续 6 个月 ≥ 生活成本的 100%。
- 平衡型：副业月收入连续 6 个月 ≥ 生活成本的 70%，且 runway ≥ 18 个月。
- 进攻型：副业月收入连续 3–6 个月 ≥ 生活成本的 50%，且 runway ≥ 24 个月，且可快速回职场。

推荐资深工程师采用平衡型起步：

$$
\text{Quit Trigger} = (\text{Side Income}_{6m} \ge 0.7 \times \text{Living Cost}) \ \land\ (\text{Runway} \ge 18)
$$

如果你有家庭责任或签证约束，把阈值提高：

$$
\text{Quit Trigger} = (\text{Side Income}_{6m} \ge 1.0 \times \text{Living Cost}) \ \land\ (\text{Runway} \ge 24)
$$

### 5.3 收入连续性的定义

“连续 6 个月”不是任意 6 个月总和达标。

它至少满足：

- 每个月都有实收款。
- 单月收入不低于目标阈值的 60%。
- 最大客户占比不超过 60%，否则单点风险过高。
- 至少有 2 个独立客户或 2 种收入来源。
- 至少 50% 收入来自可复购服务、维护、retainer、订阅或长期合同。

示例：

| 月份 | 副业实收 | 生活成本 ¥23,000 的占比 | 是否达标 | 备注 |
|---|---:|---:|---|---|
| M-6 | ¥10,000 | 43% | 否 | 一次性项目 |
| M-5 | ¥16,500 | 72% | 是 | 1 个客户 |
| M-4 | ¥18,000 | 78% | 是 | 2 个客户 |
| M-3 | ¥14,000 | 61% | 边界 | 低于 70%，但高于 60% |
| M-2 | ¥20,000 | 87% | 是 | 含维护合同 |
| M-1 | ¥19,000 | 83% | 是 | pipeline 增长 |

这组数据还不能触发平衡型离职。

原因：M-6 不达标，且 M-3 低于目标。

更稳的规则：从 M-5 重新计数，再观察 1–2 个月。

### 5.4 Side-income ramp 的三条路线

#### 路线 1：高客单咨询

适合：资深工程师、架构师、AI 工程落地经验强的人。

产品化服务示例：

- 2 周 AI 工程审计：¥20,000–¥60,000。
- RAG 原型落地：¥30,000–¥120,000。
- 内部工具自动化：¥10,000–¥50,000。
- 技术尽调：¥15,000–¥80,000。
- CTO office 顾问小时包：¥800–¥2,500/小时。

关键指标：

- 每月有效销售对话数 ≥ 8。
- 提案转化率 ≥ 25%。
- 平均客单价持续上升。
- 交付毛利率 ≥ 60%。
- 客户推荐率 ≥ 20%。

#### 路线 2：远程合同工

适合：想降低雇佣依赖，但仍需要稳定现金流的人。

可选平台与渠道：

- Toptal、Upwork、Contra、Wellfound、LinkedIn。
- 海外创业公司直接 cold email。
- 前同事、前领导、开源社区、技术社群内推。
- 国内出海团队的远程顾问需求。

关键指标：

- 时薪或日薪目标明确。
- 合同中写清 scope、payment terms、IP、termination。
- 不让单一合同占你全部时间。
- 保持每周至少 1 天用于获客与资产建设。

#### 路线 3：产品/内容复利

适合：愿意承受更长冷启动的人。

产品形式：

- 模板、插件、SaaS、Chrome extension。
- AI workflow、自动化脚本包。
- 课程、手册、newsletter、付费社群。
- 开源 + paid support。

关键指标：

- 每周发布节奏。
- 邮件列表增长。
- 免费用户到付费用户转化率。
- MRR、churn、ARPU。
- 内容带来的销售对话数。

### 5.5 决策表：何时继续在职，何时切换

| 条件 | 建议动作 | 原因 |
|---|---|---|
| Runway < 6 个月 | 不辞职，先降本增储 | 生存压力会压垮定价权 |
| Runway 6–12 个月，副业 < 30% 生活成本 | 继续在职副业 | 还在验证需求 |
| Runway 12–18 个月，副业 50%+ 连续 3 个月 | 谈远程/降工时/内部转岗 | 先灰度，不急全切 |
| Runway 18+ 个月，副业 70%+ 连续 6 个月 | 可进入离职谈判 | 已有现金流证据 |
| Runway 24+ 个月，副业 50%+，强回职场能力 | 可进攻型退出 | 用现金换时间建设资产 |
| 单一客户占副业 > 80% | 暂缓全职独立 | 这是伪独立，实为单客户雇佣 |
| 家庭/健康/签证风险高 | 提高 runway 与收入阈值 | 风险承受能力更低 |

## 6. 离职前财务去风险

离职前的关键窗口期是：你仍然有工资流水、雇主背书、稳定社保/保险、较强贷款审批能力。

不要等到辞职后才补这些。

### 6.1 建好应急金桶

执行步骤：

1. 计算必要月支出。
2. 先存 3 个月应急金。
3. 再扩到 6 个月。
4. 如果准备离职，扩到 6–12 个月，并与 runway 分开标记。
5. 应急金不要用于买课、买设备、投资加仓、创业支出。
6. 每次动用后，下一笔收入优先补回。

参考：[应急金与现金流](../investing/04-emergency-fund-cashflow.md)。

完成标准：你有一个独立账户，金额足以覆盖目标月份，且不与日常消费账户混用。

### 6.2 降低固定成本

固定成本越高，runway 越短，心理压力越大。

优先处理：

- 房租/房贷。
- 车贷。
- 高价订阅。
- 不常用 SaaS。
- 保险重复配置。
- 高利率消费贷。
- 不必要的生活方式支出。

降本不是苦行。

降本是购买选择权。

每减少 ¥1,000 月固定支出，在 ¥300,000 可动用资产下增加的 runway：

| 原月净支出 | 降低后 | 原 runway | 新 runway | 增加 |
|---:|---:|---:|---:|---:|
| ¥25,000 | ¥24,000 | 12.0 月 | 12.5 月 | 0.5 月 |
| ¥20,000 | ¥19,000 | 15.0 月 | 15.8 月 | 0.8 月 |
| ¥15,000 | ¥14,000 | 20.0 月 | 21.4 月 | 1.4 月 |
| ¥10,000 | ¥9,000 | 30.0 月 | 33.3 月 | 3.3 月 |

当支出越低，每削减 ¥1,000 的边际价值越大。

### 6.3 处理信用、贷款、房贷

如果你计划未来 12–24 个月需要信用卡、房贷、消费分期、经营贷或租房资质，尽量在离职前完成。

原因：

- 银行喜欢稳定工资流水。
- 自由职业收入证明更麻烦。
- 刚离职时你的税单、流水、合同可能不足。
- 部分地区房贷审批更看重雇佣状态。

行动清单：

- 检查信用报告。
- 关闭不必要但保留有用的信用额度。
- 还清高息债务。
- 如需房贷/再融资，在离职前咨询银行。
- 如需经营信用卡或公司账户，提前准备。
- 不为了“提高额度”而增加不必要负债。

注意：贷款会提高固定成本，不能因为更容易批就借更多。

### 6.4 医疗保险与健康连续性

健康风险是离职后最容易被低估的风险。

离职前确认：

- 当前雇主医疗保险何时终止。
- 是否有商业保险等待期。
- 社保或医保如何续缴。
- 家属是否依赖你的雇主保险。
- 慢性病、牙科、眼科、心理咨询是否需要提前处理。
- 离职后所在地的医疗支付规则。

执行建议：

1. 在离职前做一次体检。
2. 补齐必要治疗和处方药。
3. 购买或续上可承受的医疗保险。
4. 把保险保单、理赔电话、医院记录集中保存。
5. 在现金流表里单独列“healthcare”支出。

### 6.5 Vesting、RSU、期权、奖金、年假

离职时间点会影响你的真实资产。

离职前至少核对：

- 下一次 RSU vesting 日期。
- 期权 exercise window。
- 年终奖发放条件。
- 绩效奖金是否要求发放日在职。
- 未休年假是否折现。
- 离职当月社保、公积金、保险缴纳规则。
- 股票交易窗口期。
- 税务影响。

示例：

| 项目 | 日期/条件 | 金额 | 决策 |
|---|---|---:|---|
| RSU vesting | 8 月 15 日 | ¥80,000 | 如只差 3 周，通常等 vest 后再提 |
| 年终奖 | 次年 2 月发，要求在职 | ¥120,000 | 若已接近发放，评估等待价值 |
| 未休年假 | 10 天可折现 | ¥30,000 | 与 HR 确认计算方式 |
| 期权 | 离职后 90 天内行权 | 不确定 | 咨询税务与估值风险 |

不要为了“出一口气”损失一大笔确定性现金。

## 7. 法律与合同去风险

以下内容仅供教育与参考，非法律建议；雇佣合同、竞业限制、保密义务、知识产权和副业条款请咨询合格律师。

### 7.1 先读雇佣文件

准备离职或接外部客户前，收集并阅读：

- Offer letter。
- Employment agreement。
- Employee handbook。
- Confidentiality agreement。
- IP assignment agreement。
- Non-compete agreement。
- Non-solicitation agreement。
- Moonlighting / outside work policy。
- Equity plan documents。
- Bonus plan。
- Severance policy。

如果文件在公司系统里，按公司政策合法下载你有权保存的副本。

不要下载源代码、客户资料、内部文档、商业秘密或任何你无权带走的内容。

### 7.2 重点条款解释

| 条款 | 你要确认什么 | 风险 |
|---|---|---|
| Non-compete | 离职后能否加入竞争对手或做类似业务 | 影响客户与赛道选择 |
| Non-solicitation | 能否招揽前同事、客户、供应商 | 获客边界 |
| IP assignment | 工作期间或使用公司资源产生的成果归谁 | 副业产品归属 |
| Moonlighting | 在职期间能否做外部项目 | 副业合规 |
| Confidentiality | 哪些信息不能使用或披露 | 内容、咨询、案例写作风险 |
| Invention disclosure | 是否需要申报发明或项目 | 独立产品风险 |

### 7.3 副业合规边界

安全原则：

- 不使用公司电脑做副业。
- 不使用公司邮箱、Slack、VPN、代码仓库、云资源。
- 不在工作时间交付外部客户。
- 不服务明显与雇主直接竞争的客户，除非律师确认安全。
- 不把公司内部方案、客户信息、代码、文档复用到副业。
- 不公开可识别公司的敏感案例。
- 不让副业影响主业交付。

### 7.4 客户合同底线

即使是小项目，也要有书面协议。

至少写清：

- scope：做什么，不做什么。
- deliverables：交付物格式与验收标准。
- timeline：里程碑与反馈时限。
- payment：预付款、尾款、逾期处理。
- IP：交付物归属、底层工具/通用组件保留权。
- confidentiality：双方保密义务。
- liability cap：责任上限。
- termination：终止条件与已完成工作结算。
- change request：需求变更如何计费。

合同不是不信任。

合同是让关系不被记忆误差摧毁。

## 9. 如何体面辞职

体面辞职不是讨好公司。

它是保护你的长期声誉、reference、返聘可能、未来客户来源。

### 9.1 辞职时机

优先选择：

- 关键项目上线后。
- 交接文档可完成时。
- RSU/奖金/年假节点已评估后。
- 你有足够精力处理交接，而不是已经崩溃到无法沟通。
- 不在团队最混乱的当天冲动提出，除非健康或法律风险要求立即离开。

避免：

- 在会议中情绪化宣布。
- 用辞职威胁换加薪，除非你真的准备离开。
- 在没有确认合同条款前公开副业细节。
- 在社交媒体抱怨雇主或同事。

### 9.2 通知周期

常见 notice：2 周、1 个月、合同约定周期。

执行步骤：

1. 先查看合同与员工手册。
2. 准备简短书面辞职信。
3. 先约直属经理 1:1。
4. 口头沟通后发正式邮件。
5. 与经理和 HR 确认最后工作日、交接范围、设备归还、福利终止。
6. 把交接工作写进共享文档。
7. 每 2–3 天同步交接进度。

### 9.3 辞职话术模板

#### 经理 1:1 开场

```text
我想和你同步一个重要决定。经过一段时间评估，我决定离开当前全职岗位，最后工作日希望定在 YYYY-MM-DD，当然我会配合团队完成合理交接。

这个决定主要基于我接下来职业方向和生活安排的调整，不是针对团队或某个具体事件。我很感谢过去在这里的机会，也希望把交接做得专业、清楚、可持续。
```

#### 如果对方追问去向

```text
我接下来会先做一段时间的独立项目/休整/探索。具体安排还在推进中。当前我更希望把重点放在交接，确保团队不受影响。
```

#### 如果对方挽留

```text
谢谢你愿意挽留，我很珍惜这份认可。这个决定我已经综合考虑了财务、家庭和长期方向。如果团队有过渡需要，我愿意讨论一个清晰边界的交接方案，甚至在离职后以合同形式支持一段时间。
```

#### 辞职邮件

```text
Subject: Resignation Notice

Hi <Manager>,

As discussed, I am formally submitting my resignation from my role as <Role>. My proposed last working day is <Date>, in line with the required notice period.

I appreciate the opportunities and support I have received during my time here. Over the transition period, I will document my current responsibilities, hand over active projects, and support the team to make the transition as smooth as possible.

Please let me know the next steps with HR, equipment return, benefits, and any required paperwork.

Best,
<Name>
```

### 9.4 离职期间不要做的事

- 不删除或破坏资料。
- 不带走不属于你的文件。
- 不拉同事吐槽公司。
- 不在交接中故意留坑。
- 不公开内部信息。
- 不承诺无限期免费支持。
- 不把个人情绪写进正式邮件。

### 9.5 保持桥梁

离职最后一周做三件事：

1. 给核心同事写感谢信息。
2. 给经理交付 handover index。
3. 给愿意保持联系的人留下个人邮箱或 LinkedIn。

感谢信息示例：

```text
这段时间和你合作我学到很多，尤其是 <具体项目/具体能力>。我之后会用个人邮箱 <email> 保持联系。如果未来你需要 <你的专业方向> 方面的外部支持，也欢迎随时找我。
```

## 10. 谈判 exit / transition

离职不一定是二元开关。

你可以谈一个过渡结构。

### 10.1 可谈选项

| 选项 | 适合情况 | 你得到什么 | 公司得到什么 |
|---|---|---|---|
| 延长交接 | 系统复杂、替代者未到 | 更好关系、可能补偿 | 降低交接风险 |
| Contract-back | 公司仍需要你的上下文 | 离职后现金流 | 按需获取专家支持 |
| Part-time | 团队预算有限但仍需支持 | 时间自由 + 稳定收入 | 保留关键能力 |
| Advisory retainer | 只需架构评审/答疑 | 高时薪、低占用 | 避免重大决策失误 |
| Project-based | 明确迁移或上线任务 | 可控交付范围 | 结果导向 |

### 10.2 Contract-back 谈判原则

不要从“我可以继续帮忙”开始。

要从“如何降低交接风险”开始。

你可以提出：

- 每周最多 X 小时。
- 固定时段可用。
- 响应 SLA。
- 明确 hourly rate 或 retainer。
- 工作范围仅限特定系统。
- 不参与日常管理和无限会议。
- 合同期限 1–3 个月，到期再评估。

话术：

```text
如果团队担心某些系统在我离开后有上下文缺口，我可以讨论一个离职后的短期顾问安排。

例如在 8 周内，每周最多 6 小时，主要覆盖 <系统/模块> 的架构答疑、故障复盘和关键 PR review。费用可以按 hourly rate 或 monthly retainer 计算。这样团队有稳定过渡窗口，我也能清晰控制时间边界。
```

### 10.3 费用怎么定

粗略公式：

$$
\text{顾问时薪}=\frac{\text{原年总包}}{\text{年工作小时}}\times\text{风险与自雇倍率}
$$

其中：

- 年工作小时可用 1,600–1,800 小时。
- 自雇倍率常见 1.5–3 倍。
- 如果是紧急、关键系统、稀缺上下文，可更高。

示例：

| 原年总包 | 年工作小时 | 等效雇员时薪 | 2x 顾问价 | 3x 顾问价 |
|---:|---:|---:|---:|---:|
| ¥600,000 | 1,700 | ¥353/h | ¥706/h | ¥1,059/h |
| ¥900,000 | 1,700 | ¥529/h | ¥1,058/h | ¥1,587/h |
| ¥1,200,000 | 1,700 | ¥706/h | ¥1,412/h | ¥2,118/h |

不要用原工资时薪做顾问价。

自雇需要覆盖：

- 获客时间。
- 税务与会计。
- 保险。
- 设备与软件。
- 无薪假期。
- 项目间空档。
- 法律风险。

### 10.4 过渡合同要写清

- 起止日期。
- 每周或每月最多小时数。
- 费用与付款周期。
- 是否预付。
- 响应时间。
- 工作范围。
- 不包含的事项。
- 保密义务。
- IP 与工作成果归属。
- 终止条款。
- 责任上限。

再次提醒：合同条款请咨询专业律师。

## 11. 风险场景与 contingency plan

离职前写 contingency plan，会让你更敢执行。

因为你知道每种坏情况的下一步。

### 11.1 风险场景表

| 场景 | 早期信号 | 预防动作 | 触发后动作 |
|---|---|---|---|
| 副业收入断流 | pipeline 连续 4 周无新对话 | 每周固定获客，保留 2+ 渠道 | 降本、启动 cold outreach、接短合同 |
| 最大客户流失 | 单客户收入占比 > 60% | 限制客户占比，增加 retainer | 立即扩展 20 个潜在客户触达 |
| 项目延期收款 | 客户反馈慢、验收模糊 | 里程碑付款、预付款 | 停止新增工作，按合同催款 |
| 健康问题 | 睡眠下降、持续疼痛 | 保险、体检、运动 | 启用医疗预算，减少交付负荷 |
| 家庭支出增加 | 家人失业、照护需求 | 家庭应急金 | 调整 runway，暂停高风险项目 |
| 市场下行 | 招聘减少、客户预算冻结 | 多行业客户、低价产品线 | 转向刚需降本类服务 |
| 法律纠纷 | 客户 scope creep、IP 争议 | 合同、邮件留痕 | 暂停交付，找律师 |
| 心理崩溃 | 拖延、昼夜颠倒、社交隔离 | 作息、同伴、教练 | 降低目标，寻求专业支持 |

### 11.2 收入断流 30 天应急流程

第 1 天：确认现金位置。

- 更新资产表。
- 计算 no-income runway。
- 冻结非必要大额支出。
- 列出所有应收账款。

第 2–3 天：恢复销售动作。

- 联系过去客户。
- 联系前同事和前经理。
- 发布明确服务说明。
- 每天发出 10 个高质量触达。

第 4–7 天：设计短周期产品。

- 选择 1 个 1–2 周可交付服务。
- 设置固定价格。
- 写明交付物。
- 找 5 个目标客户验证。

第 2 周：扩大渠道。

- LinkedIn / Twitter / 即刻 / 技术社区发布案例。
- 参加 2 场行业活动。
- 联系 3 个互补服务商做 referral。
- 申请远程合同岗位。

第 3–4 周：现金优先。

- 接受一个合规、边界清晰、现金流好的项目。
- 不接低信任、高风险、无限 scope 的项目。
- 每周复盘报价、转化、拒绝原因。

### 11.3 健康问题 contingency

如果出现健康问题，优先级从“增长”切换到“恢复生存能力”。

行动：

1. 立即减少交付承诺。
2. 通知客户调整时间线，保留书面记录。
3. 使用医疗保险或应急金。
4. 暂停高认知负荷获客。
5. 把工作切到低风险维护、文档、轻量咨询。
6. 如果 30–60 天无法恢复，启动 re-entry plan 或寻找兼职稳定收入。

### 11.4 Re-entry plan：让离职不是不可逆

安全离职不等于永不回职场。

真正的自由是你能选择回来，也能选择再出去。

离职前准备：

- 更新简历。
- 更新 LinkedIn。
- 整理 5 个 STAR 故事。
- 整理 3 个系统设计案例。
- 保留 10–20 个高质量职场联系人。
- 每季度与 3–5 位前同事保持联系。
- 关注远程岗位和合同岗位市场。
- 保持代码、写作、案例输出。

触发 re-entry 的条件：

| 条件 | 动作 |
|---|---|
| Conservative runway < 9 个月 | 开始投递合同工与全职远程岗位 |
| Conservative runway < 6 个月 | 求职优先级升到 P0 |
| 连续 3 个月收入 < 生活成本 30% | 暂停长期产品，优先现金流 |
| 健康问题持续影响交付 60 天 | 寻找低压稳定收入或休养方案 |
| 家庭责任显著增加 | 重新计算风险承受能力 |

求职不代表失败。

它只是你的回滚方案被触发。

## 12. 6–12 个月过渡计划

下面是一份可执行计划。

你可以根据自己的 runway 和副业成熟度压缩或拉长。

### Month 1：基线审计

目标：知道自己真实位置。

动作：

- 建立资产、支出、收入、pipeline 四张表。
- 计算 emergency fund、runway、F-you money 等级。
- 读完雇佣合同和副业政策。
- 列出所有固定成本。
- 写出离职原因，但不做离职动作。
- 选择 1 条副业主线。

指标：

- 保守 runway 数字可解释。
- 支出分类覆盖 90%+。
- 副业主线一句话能说清。
- 法律风险初步标记完成。

产出物：

- `runway.xlsx` 或等价表格。
- 一页“退出假设”。
- 合同风险问题清单。

### Month 2：降本与应急金

目标：降低 burn rate。

动作：

- 削减 3–5 项订阅或固定支出。
- 把应急金补到至少 3–6 个月。
- 检查信用报告和贷款需求。
- 安排体检。
- 明确保险续接方案。

指标：

- 月固定成本下降 5–15%。
- 应急金账户独立。
- 保险中断风险已关闭。

产出物：

- 降本记录。
- 医疗/保险清单。
- 信用与贷款决策记录。

### Month 3：副业 offer 验证

目标：市场愿意为什么付钱。

动作：

- 写一页服务说明。
- 设计 1 个固定范围服务包。
- 找 20 个目标客户访谈或触达。
- 完成 3–5 次销售对话。
- 争取第一个付费试点。

指标：

- 有效回复率 ≥ 10%。
- 销售对话 ≥ 3。
- 至少 1 个付费或强意向。

产出物：

- 服务说明页。
- CRM 表。
- 报价模板。

### Month 4：交付与案例

目标：把一次收入变成可复用案例。

动作：

- 交付第一个项目。
- 写交付复盘。
- 提炼 SOP。
- 请求 testimonial。
- 把案例匿名化公开。

指标：

- 项目按范围交付。
- 客户满意度 ≥ 8/10。
- 实收款到账。
- 案例可用于下一轮获客。

产出物：

- 匿名案例。
- 交付 checklist。
- 客户推荐语。

### Month 5：重复获客

目标：证明不是偶然成交。

动作：

- 每周固定 5–10 个高质量触达。
- 每周发布 1 篇技术/案例内容。
- 维护 referral 网络。
- 提高报价或收预付款。
- 记录每个 lead 的来源。

指标：

- 月销售对话 ≥ 6。
- 提案 ≥ 2。
- 新成交 ≥ 1。
- pipeline value ≥ 未来 2 个月生活成本。

产出物：

- 获客周报。
- 内容发布记录。
- pipeline dashboard。

### Month 6：第一次离职评估

目标：决定继续在职、副业加速、谈降工时，还是准备离职。

动作：

- 复算 runway。
- 统计最近 6 个月副业实收。
- 检查客户集中度。
- 检查健康和家庭压力。
- 对照决策表。

指标：

- Runway ≥ 12 个月才进入下一步讨论。
- 副业收入 ≥ 生活成本 50% 才考虑灰度。
- 客户来源至少 2 个。

决策：

| 结果 | 动作 |
|---|---|
| 未达标 | 继续在职 3 个月，优化 offer |
| 部分达标 | 谈远程/降工时/内部转岗 |
| 达标 | 准备 Month 7–9 离职窗口 |

### Month 7：谈判灰度

目标：减少主业占用而不断现金流。

动作：

- 评估能否转远程、四天工作制、内部顾问或降工时。
- 如果不可行，准备正式离职计划。
- 完成 handover 草稿。
- 明确 contract-back 方案。

指标：

- 每周副业可用时间增加。
- 主业交付不失控。
- 过渡谈判选项清晰。

产出物：

- 降工时提案或离职计划。
- Handover index。
- Contract-back proposal。

### Month 8：离职窗口准备

目标：关闭离职前风险。

动作：

- 确认 RSU、奖金、年假节点。
- 确认保险续接。
- 咨询合同法律风险。
- 准备辞职邮件和 1:1 话术。
- 把 pipeline 覆盖到未来 3 个月。

指标：

- 保守 runway ≥ 18 个月。
- 副业连续性满足你的触发规则。
- 离职检查清单完成 80%+。

产出物：

- 最终离职决策记录。
- 风险清单。
- 辞职脚本。

### Month 9：正式提出离职

目标：体面退出并谈过渡。

动作：

- 与经理 1:1。
- 发正式辞职邮件。
- 与 HR 确认流程。
- 执行 handover。
- 讨论 contract-back 或 reference。

指标：

- 最后工作日确认。
- 交接 owner 明确。
- 福利终止日期明确。
- 过渡合同是否可行有结论。

产出物：

- Handover 文档。
- HR checklist。
- 过渡合同草案。

### Month 10：离职后第一个月

目标：不让自由变成混乱。

动作：

- 固定每周计划节奏。
- 每周至少 2 天获客。
- 每周至少 2 天交付。
- 每周 1 次财务更新。
- 不在第一个月做大额 lifestyle spending。

指标：

- Burn rate ≤ 计划值。
- Sales conversations ≥ 8/月。
- 实收款按计划到账。
- 睡眠与运动稳定。

产出物：

- 每周复盘。
- 月度 P&L。
- 下月 pipeline。

### Month 11：系统化与提价

目标：从“接活”升级为“业务系统”。

动作：

- 把重复交付写成模板。
- 提高最低报价。
- 设置客户筛选问题。
- 建立 referral request 流程。
- 开始建设长期资产：内容、产品、开源、newsletter。

指标：

- 平均客单价提升。
- 无效会议减少。
- Referral lead 增加。
- 可复用模板 ≥ 3 个。

产出物：

- SOP 库。
- 价格页。
- 客户筛选表。

### Month 12：年度复盘与继续/回撤决策

目标：决定继续独立、扩大、转合同工，或回职场。

动作：

- 汇总 12 个月收入、支出、税务、健康、满意度。
- 比较预期与实际。
- 更新 runway。
- 判断是否触发 re-entry。
- 设定下一年度策略。

指标：

- 年化收入趋势。
- Runway 是否下降过快。
- 客户集中度。
- 工作时长。
- 身心状态。

决策：

| 状态 | 下一步 |
|---|---|
| 收入稳定且 runway 不降 | 继续独立，投资系统化 |
| 收入增长但过劳 | 提价、减少客户、产品化 |
| 收入不足但 pipeline 强 | 再给 3–6 个月，但降本 |
| 收入不足且 runway < 9 月 | 启动 re-entry plan |
| 健康明显变差 | 降载，优先恢复 |

## 13. 最小可执行版本：7 天内完成离职安全审计

如果你现在已经很想辞职，先不要发邮件。

用 7 天完成最小审计。

### Day 1：现金盘点

- 列出所有现金和低波动资产。
- 标记可用时间。
- 对波动资产打折。
- 得出可动用流动资产。

完成标准：一个保守资产总数。

### Day 2：支出盘点

- 导出最近 3–6 个月账单。
- 分成必要、可削减、可暂停。
- 估算离职后必要支出。
- 找出 3 个立刻可降的固定成本。

完成标准：一个月净支出数字。

### Day 3：Runway 计算

- 计算无收入 runway。
- 计算保守副业收入口径 runway。
- 计算最坏情况：额外支出 + 收入为 0。
- 写下目标 runway。

完成标准：知道自己离目标差几个月或多少钱。

### Day 4：合同风险

- 找出雇佣合同。
- 标记竞业、保密、IP、副业条款。
- 写下要咨询律师的问题。
- 停止任何可能违规的副业行为。

完成标准：风险不是一团雾。

### Day 5：副业与 pipeline

- 列出过去所有付费项目。
- 列出 20 个潜在客户或 referral。
- 写一页服务说明。
- 发出 5 个低风险触达。

完成标准：市场验证开始，而不是只想象。

### Day 6：福利与健康

- 查保险终止日期。
- 查社保/医保续缴方式。
- 查 RSU/奖金/年假节点。
- 预约体检或处理已有健康问题。

完成标准：知道离职会切断哪些保障。

### Day 7：决策备忘录

写一页 memo：

```text
我为什么想离职：

当前 runway：
目标 runway：
无收入 runway：
副业连续性：
最大风险：
离职触发条件：
不离职的替代动作：
最早可提离职日期：
回撤触发条件：
```

完成标准：你能把“我想走”变成“满足这些条件我就走”。

## 14. 常见坑与排查表

| 坑 | 症状 | 解决办法 |
|---|---|---|
| 把税前收入当可花收入 | 账上看似赚钱，年底补税痛苦 | 每笔收入先预留税费 |
| 把开票当收入 | 客户拖款导致现金断裂 | 只按实收款计算 runway |
| 把一次性项目当 MRR | 一个月很好，下个月归零 | 区分 project revenue 与 recurring revenue |
| 没有合同 | scope creep、付款争议 | 小项目也写 SOW |
| 客户太集中 | 一个客户取消就崩 | 单客户占比控制在 50–60% 以下 |
| 离职前大额消费 | runway 被生活方式吞掉 | 离职前 6 个月冻结非必要大额支出 |
| 没读竞业/IP | 后续客户或产品受限 | 提前找律师审条款 |
| 没有作息 | 自由后效率下降 | 用周计划和日复盘接管时间 |
| 只做交付不获客 | 项目结束后无收入 | 每周固定获客时段 |
| 用低价换安全感 | 越忙越穷 | 设置最低价格和拒绝条件 |
| 离职时烧桥 | reference 和返聘机会消失 | 专业交接，感谢关键同事 |
| 没有回职场方案 | runway 下降时恐慌 | 离职前写 re-entry plan |

## 16. 离职后 30 天操作系统

离职后的第一个月，不要把时间全部用于“恢复”。

可以休息，但必须有结构。

### 每周节奏

| 时间块 | 建议安排 |
|---|---|
| 周一上午 | 财务、pipeline、周目标 |
| 周一下午 | 销售触达 |
| 周二 | 客户交付 |
| 周三上午 | 内容/案例/资产建设 |
| 周三下午 | 销售对话 |
| 周四 | 客户交付 |
| 周五上午 | 复盘、开票、催款 |
| 周五下午 | 学习与系统优化 |
| 周末 | 休息、运动、家庭、低压阅读 |

### 每周指标

- 新增有效 lead ≥ 5。
- 销售对话 ≥ 2。
- Proposal ≥ 1。
- 内容或案例输出 ≥ 1。
- 实收款状态更新 1 次。
- Burn rate 更新 1 次。
- 运动 ≥ 3 次。
- 睡眠均值 ≥ 7 小时。

### 每周复盘问题

```text
本周现金流：
本周新增 pipeline：
本周交付进度：
本周最浪费时间的事情：
本周最有效的获客动作：
下周必须完成的 3 件事：
是否触发任何风险阈值：
```

把这些接入 [Solo Productivity System](03-solo-productivity-system.md)，避免离职后靠意志力管理自己。

## 18. 最终检查：做到这些再提离职

- [ ] 我能用一句话说明离职不是逃避，而是系统迁移。
- [ ] 我已用公式算出保守 runway 和无收入 runway。
- [ ] 我的目标 runway 是 12、18 还是 24 个月，并能解释原因。
- [ ] 我已建立应急金，且阅读过 [应急金与现金流](../investing/04-emergency-fund-cashflow.md)。
- [ ] 我已对照 [FIRE 基础](../investing/01-fire-basics.md) 区分 F-you money 与长期财务自由。
- [ ] 我已选择至少一条副业路线，并参考 [变现地图](../income/01-monetization-landscape.md)。
- [ ] 我已完成 6–12 个月过渡计划的当前月份动作。
- [ ] 我已写好离职触发条件。
- [ ] 我已写好 re-entry plan。
- [ ] 我已准备体面辞职脚本。
- [ ] 我已准备交接文档。
- [ ] 我已知道哪些法律和合同问题需要专业意见。
- [ ] 我已把离职后的周节奏接入 [Solo Productivity System](03-solo-productivity-system.md)。

如果这些没有完成，最好的下一步通常不是辞职。

最好的下一步是把系统补齐。

安全离职不是胆小。

安全离职是把自由从情绪愿望变成可持续资产。
