# 应急金与现金流管理（投资的地基）

> 结论先行：**没有应急金的投资是在流沙上盖楼**。一次失业、医疗、家庭意外或现金流断档，就可能逼你在市场低点卖出长期资产。先建好现金缓冲、理顺现金流，投资才谈得上“长期持有”。而真正决定你多快接近 FIRE 的，不只是收益率，更是**稳定、可持续的储蓄率**。

> **免责声明：本文仅用于教育用途，非投资建议。** 不同国家/地区的存款保障、货币基金规则、税制、保险制度、债券市场和账户类型差异很大；请结合所在地法规、个人负债、家庭责任和风险承受能力做决策。

相关阅读：[FIRE 基础](01-fire-basics.md)、[指数基金投资指南](03-index-investing-guide.md)、[资产配置与再平衡](02-asset-allocation-rebalancing.md)。

## 本章定位：先保证“不断粮”，再谈“滚雪球”

应急金与现金流管理是个人投资体系的地基。它不追求最高收益，而追求三件事：

1. **危机时不被迫卖出长期资产**：股票、指数基金、长期债券可能在你最需要钱时下跌。
2. **日常支出有秩序**：收入、账单、储蓄、投资和目标资金各走各的通道。
3. **储蓄率可被观测、可被改进**：你能像看系统监控一样看自己的家庭现金流。

学完后，你应该能独立完成：

- 计算自己的“月必要开支”和应急金目标金额。
- 按工作稳定性、家庭责任、收入波动选择 3/6/12 个月缓冲。
- 比较 HYSA、money market、T-bills、货币基金、活期/定期存款的流动性与收益取舍。
- 用 50/30/20、zero-based budgeting、sinking funds 管理现金流。
- 搭建一个可复制的月度现金流表格，并用公式计算储蓄率、缺口、目标进度。
- 每月用 30 分钟完成一次财务 review。

## 1. 核心心智模型：应急金是“财务熔断器”

把个人资产负债表想成一个线上服务：

| 系统概念 | 财务对应物 | 目标 |
| --- | --- | --- |
| 请求流量 | 每月支出 | 可预测、可限流 |
| 稳定收入 | 主服务实例 | 持续处理现金需求 |
| 应急金 | 熔断器 + 缓存 | 主服务挂掉时顶住流量 |
| 保险 | 灾备机制 | 处理低频高损失事件 |
| 长期投资 | 后台增长任务 | 不被短期故障打断 |
| 月度复盘 | observability | 发现异常并修正 |

应急金不是“收益率低的闲钱”，而是一个独立功能模块：

```text
收入到账
  ↓
先付给自己：储蓄 / 应急金 / 投资
  ↓
固定必要支出：房租、房贷、保险、水电、交通、基本饮食
  ↓
目标支出：旅行、换手机、教育、年度账单（sinking funds）
  ↓
自由支配：娱乐、购物、升级消费
```

没有应急金时，系统常退化成：

```text
突发支出 → 信用卡/高息贷款 → 利息滚动 → 停止投资 → 被迫卖出资产 → 长期计划中断
```

有应急金后，链路变成：

```text
突发支出 → 使用现金缓冲 → 暂停新增投资或降低可选消费 → 补回应急金 → 长期资产不动
```

关键差异不是“多赚了多少利息”，而是**避免在坏时间做坏决策**。

## 2. 第一步：算出“月必要开支”

应急金的单位不是“月收入”，而是“月必要开支”。收入高但固定支出也高的人，需要的缓冲可能更大。

### 2.1 必要开支的定义

| 类别 | 包含 | 是否计入应急金基数 |
| --- | --- | --- |
| 住房 | 房租、房贷、物业、必要维修 | 是 |
| 基础食物 | 买菜、基本餐饮 | 是 |
| 水电通信 | 水、电、燃气、手机、宽带 | 是 |
| 交通 | 通勤、油费、停车、公共交通 | 是 |
| 保险 | 医疗、意外、车险、定期寿险 | 是 |
| 医疗 | 常规药品、复诊、慢病支出 | 是 |
| 债务最低还款 | 信用卡最低还款、学生贷、车贷 | 是 |
| 子女/赡养 | 托育、学费、父母基础支持 | 是 |
| 投资定投 | 指数基金、养老金额外缴纳 | 通常否，危机时可暂停 |
| 娱乐消费 | 游戏、购物、旅行、会员升级 | 否 |

### 2.2 三种口径

| 口径 | 用途 | 计算方式 |
| --- | --- | --- |
| 当前生活口径 | 观察真实支出 | 最近 3–6 个月平均总支出 |
| 必要开支口径 | 设定应急金基数 | 删除可暂停消费后的月支出 |
| 紧急模式口径 | 极端情况下续航 | 只保留生存、住房、保险、最低还款 |

示例：

| 项目 | 当前生活 | 必要开支 | 紧急模式 |
| --- | ---: | ---: | ---: |
| 房租/房贷 | 12,000 | 12,000 | 12,000 |
| 食物 | 5,000 | 4,000 | 3,000 |
| 水电通信 | 1,200 | 1,200 | 1,000 |
| 交通 | 2,000 | 1,500 | 1,000 |
| 保险 | 1,500 | 1,500 | 1,500 |
| 医疗 | 800 | 800 | 800 |
| 债务最低还款 | 2,000 | 2,000 | 2,000 |
| 子女/赡养 | 3,000 | 3,000 | 3,000 |
| 娱乐/购物 | 5,000 | 1,000 | 0 |
| 旅行/大额消费 | 3,000 | 0 | 0 |
| **合计** | **35,500** | **27,000** | **24,300** |

这位家庭的应急金基数应使用 27,000，而不是月收入，也不是 35,500。

### 2.3 快速计算公式

```text
月必要开支 = 住房 + 基础食物 + 水电通信 + 必要交通 + 保险 + 医疗 + 债务最低还款 + 家庭责任
应急金目标 = 月必要开支 × 目标月数
现金缺口 = 应急金目标 - 当前可用应急金余额
预计建成月数 = 现金缺口 ÷ 每月可投入应急金金额
```

如果刚搬家、换城市、结婚、生子，应使用未来 3–6 个月的预算估算，而不是旧数据。

## 3. 第二步：决定留 3、6、9 还是 12 个月

“3–6 个月”是常见经验值，但不是铁律。应急金月数取决于两个变量：收入恢复速度、支出刚性。

| 情况 | 建议月数 | 理由 |
| --- | ---: | --- |
| 单身、无负债、技能稀缺、行业稳定 | 3–4 个月 | 支出弹性大，收入恢复快 |
| 双职工、无高息债、家庭支出稳定 | 4–6 个月 | 两份收入降低断档概率 |
| 单收入家庭、有房贷/车贷/子女 | 6–9 个月 | 支出刚性高，容错空间要大 |
| 行业周期强、岗位集中、签证/身份受雇主影响 | 9–12 个月 | 找工作周期可能显著拉长 |
| 自由职业、创业、销售提成占比高 | 9–12 个月 | 收入本身波动大 |
| 家中有慢性病、老人赡养、特殊教育支出 | 9–12 个月 | 不可暂停支出多 |
| 已接近退休或 FIRE 过渡期 | 12–24 个月现金/短债桶 | 需要降低序列风险 |

### 3.1 打分法

| 因素 | 0 分 | 1 分 | 2 分 |
| --- | --- | --- | --- |
| 收入来源 | 双收入且稳定 | 单收入但稳定 | 单收入且波动大 |
| 行业/岗位 | 需求稳定 | 有周期波动 | 裁员频繁/岗位稀缺 |
| 负债 | 无高息债、低固定支出 | 有房贷/车贷 | 高固定支出或高息债 |
| 家庭责任 | 单身/无依赖 | 伴侣或父母部分依赖 | 子女/老人高度依赖 |
| 医疗风险 | 保障充分 | 有常规医疗支出 | 高额或不确定医疗支出 |
| 流动资产 | 现金充足 | 可卖资产但会波动 | 资产锁定或难变现 |

| 总分 | 建议目标 |
| ---: | --- |
| 0–3 | 3–4 个月 |
| 4–6 | 6 个月 |
| 7–9 | 9 个月 |
| 10–12 | 12 个月或更多 |

### 3.2 数字例子 A：单身工程师

假设税后月收入 45,000，月必要开支 18,000，当前现金 40,000，无高息债，每月可投入应急金 12,000。

```text
目标应急金 = 18,000 × 4 = 72,000
现金缺口 = 72,000 - 40,000 = 32,000
预计建成 = 32,000 ÷ 12,000 = 2.67 个月
```

完成标准：3 个月内把专用应急金账户补到 72,000，之后新增储蓄可转向长期投资。

### 3.3 数字例子 B：单收入三口之家

假设税后月收入 55,000，月必要开支 32,000，当前现金 60,000，房贷 + 子女支出 + 单收入，每月可投入应急金 8,000。

```text
目标应急金 = 32,000 × 9 = 288,000
现金缺口 = 288,000 - 60,000 = 228,000
预计建成 = 228,000 ÷ 8,000 = 28.5 个月
```

分阶段执行：

1. 先补到 1 个月：32,000，已经完成。
2. 补到 3 个月：96,000，再需要 36,000，约 5 个月。
3. 3 个月缓冲达成后，可把新增现金拆分：70% 补应急金，30% 投资。
4. 达到 9 个月后，再提高长期投资比例。

## 4. 第三步：选择存放地点：安全、流动、收益的三角取舍

应急金的首要目标是：**在需要时能按面值或接近面值快速拿到钱**。

| 工具 | 适合放哪一层 | 流动性 | 本金波动 | 收益特征 | 注意事项 |
| --- | --- | --- | --- | --- | --- |
| 活期存款/checking | 0.5–1 个月 | 最高 | 通常无 | 最低 | 用于账单和即时支出 |
| 高收益储蓄账户 HYSA | 1–6 个月 | 高 | 通常无 | 随利率变化 | 确认存款保险和提现限制 |
| money market deposit account | 1–6 个月 | 高 | 通常无 | 接近短端利率 | 与 money market fund 不同 |
| money market fund | 1–6 个月 | 高 | 极低但非零 | 接近短端利率 | 不是银行存款，极端情况下可能波动 |
| 货币基金 | 1–6 个月 | 高 | 低但非零 | 通常高于活期 | 查看赎回到账时间与限额 |
| 短期国债/T-bills | 第 2 层或第 3 层 | 中高 | 持有到期通常确定 | 与短端利率相关 | 到期日、税务、提前卖出价格 |
| 1–3 个月定期/大额存单 | 第 3 层 | 中 | 通常低 | 锁定利率 | 提前支取规则可能损失利息 |
| 股票/股票基金 | 不适合 | 高 | 高 | 长期更高 | 急用时可能亏损 |
| 长久期债券基金 | 不适合核心应急金 | 高 | 中高 | 对利率敏感 | 利率上行时可能下跌 |
| 锁定期理财/私募/房产 | 不适合 | 低 | 不确定 | 不确定 | 危机时难变现 |

### 4.1 推荐分层

| 层级 | 金额 | 放置位置 | 用途 |
| --- | ---: | --- | --- |
| 第 0 层：账单缓冲 | 0.5 个月必要开支 | 工资卡/checking/活期 | 防止扣款失败、日常周转 |
| 第 1 层：即时应急 | 1–2 个月必要开支 | HYSA/货币基金/高流动账户 | 失业、维修、医疗垫付 |
| 第 2 层：核心缓冲 | 2–6 个月必要开支 | HYSA、money market、短期限 T-bills 梯子 | 支撑较长断档 |
| 第 3 层：扩展缓冲 | 6–12 个月以上 | T-bills 梯子、短定期、短债 | 自由职业、单收入家庭、接近退休 |

例子：月必要开支 30,000，目标 6 个月，即 180,000。

| 层级 | 金额 | 示例配置 |
| --- | ---: | --- |
| 第 0 层 | 15,000 | 活期/工资卡 |
| 第 1 层 | 45,000 | HYSA 或货币基金 |
| 第 2 层 | 90,000 | HYSA + money market fund |
| 第 3 层 | 30,000 | 4 周或 13 周 T-bills 梯子 |
| **合计** | **180,000** | 现金与短期工具 |

### 4.2 T-bills 梯子的简单做法

把 60,000 拆成 4 份，每份 15,000：

| 批次 | 买入工具 | 到期节奏 | 目的 |
| --- | --- | --- | --- |
| A | 4 周 T-bill | 第 4 周到期 | 第一笔回流 |
| B | 8 周 T-bill | 第 8 周到期 | 第二笔回流 |
| C | 13 周 T-bill | 第 13 周到期 | 第三笔回流 |
| D | 13 周 T-bill | 第 17 周附近再滚动 | 维持梯子 |

维护规则：不把第 0 层和第 1 层放入 T-bills；到期先检查是否动用应急金；如需现金则不续买；不要为了多一点收益买长期债券。

### 4.3 产品检查清单

- [ ] 本金是否有政府存款保险或等价保护？额度是多少？
- [ ] 是否可能出现净值波动？历史最大回撤是多少？
- [ ] 赎回到账是实时、T+1、T+2，还是更久？
- [ ] 单日/单笔赎回限额是多少？
- [ ] 提前支取是否损失本金或大量利息？
- [ ] 收益是否需要纳税？税后收益是多少？
- [ ] 是否有管理费、申购费、赎回费、账户维护费？
- [ ] 家人是否知道紧急情况下如何取用？

## 5. 第四步：建立应急金的执行路径

有余钱时按这个顺序处理，通常效率最高：

```text
1. 先攒 1 个月小额缓冲（防小意外）
2. 还清高息负债（信用卡 / 高息消费贷）
3. 补满 3 个月基础应急金
4. 用足雇主匹配 / 税优账户中“确定性收益”部分（若适用）
5. 补到你的目标应急金月数（6 / 9 / 12 个月）
6. 长期指数投资（见 03-index-investing-guide.md）
7. 其它目标：教育、购房、创业、提高风险资产比例
```

高息债的优先级很高，因为：

```text
还掉 18% 年化信用卡债 ≈ 获得 18% 无风险税后收益
```

### 5.1 分阶段目标

| 阶段 | 目标 | 完成标准 | 下一步 |
| --- | --- | --- | --- |
| Stage 0 | 不再透支 | 账单不靠信用卡滚动 | 建 1 个月缓冲 |
| Stage 1 | 1 个月必要开支 | 任意小意外不打断生活 | 清高息债 |
| Stage 2 | 无高息债 | 信用卡全额还款，无消费贷滚动 | 建 3 个月缓冲 |
| Stage 3 | 3 个月应急金 | 可支撑短期失业 | 开始小比例长期投资 |
| Stage 4 | 目标月数 | 达到 6/9/12 个月 | 提高长期投资比例 |
| Stage 5 | 自动维护 | 动用后自动补回 | 年度复核目标 |

### 5.2 自动转账规则

1. 发薪日当天或次日自动转账到“应急金账户”。
2. 转账金额设为固定数，例如 5,000、8,000、12,000。
3. 如果收入浮动，使用公式：

```text
本月应急金转入 = max(固定最低额, 税后收入 × 目标储蓄率 × 应急金分配比例)
```

例子：

```text
税后收入 = 42,000
目标储蓄率 = 30%
应急金分配比例 = 70%
本月应急金转入 = 42,000 × 30% × 70% = 8,820
```

4. 转账后再支付房租、账单和日常消费。
5. 不要把应急金账户绑定日常消费卡，降低误用概率。

### 5.3 应急金 build-up schedule 示例

假设：月必要开支 25,000；目标 6 个月；目标金额 150,000；当前余额 30,000；每月新增 10,000；年化收益假设 2.4%，即月收益约 0.2%。

| 月份 | 期初余额 | 本月转入 | 估算利息 | 期末余额 | 覆盖月数 | 状态 |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| 0 | 30,000 | 0 | 0 | 30,000 | 1.20 | 已有缓冲 |
| 1 | 30,000 | 10,000 | 80 | 40,080 | 1.60 | 继续 |
| 2 | 40,080 | 10,000 | 100 | 50,180 | 2.01 | 继续 |
| 3 | 50,180 | 10,000 | 120 | 60,300 | 2.41 | 达到 2+ 月 |
| 4 | 60,300 | 10,000 | 141 | 70,441 | 2.82 | 继续 |
| 5 | 70,441 | 10,000 | 161 | 80,602 | 3.22 | 达到 3 月，可小比例投资 |
| 6 | 80,602 | 10,000 | 181 | 90,783 | 3.63 | 继续 |
| 7 | 90,783 | 10,000 | 202 | 100,985 | 4.04 | 继续 |
| 8 | 100,985 | 10,000 | 222 | 111,207 | 4.45 | 继续 |
| 9 | 111,207 | 10,000 | 242 | 121,449 | 4.86 | 继续 |
| 10 | 121,449 | 10,000 | 263 | 131,712 | 5.27 | 继续 |
| 11 | 131,712 | 10,000 | 283 | 141,995 | 5.68 | 接近目标 |
| 12 | 141,995 | 8,005 | 300 | 150,300 | 6.01 | 达标 |

公式：

```text
估算利息 = (期初余额 + 本月转入) × 月利率
期末余额 = 期初余额 + 本月转入 + 估算利息
覆盖月数 = 期末余额 ÷ 月必要开支
```

收益是假设，不是承诺。应急金计划应在不依赖收益的情况下也能完成。

## 6. 现金流管理：把钱分配到正确的“队列”

现金流管理不是为了控制每一杯咖啡，而是为了保证：

1. 必要支出永远有钱。
2. 应急金按计划增长。
3. 长期投资不会因为年度账单被迫中断。
4. 大额目标提前准备，不靠临时借债。
5. 自由消费有边界，不产生内疚感。

### 6.1 关键指标

| 指标 | 公式 | 健康区间 | 用途 |
| --- | --- | --- | --- |
| 储蓄率 | 月储蓄 / 税后收入 | 20%+ 起步，FIRE 通常更高 | 决定财富积累速度 |
| 必要支出率 | 必要支出 / 税后收入 | 越低越灵活 | 衡量抗风险能力 |
| 固定支出率 | 固定账单 / 税后收入 | 高于 60% 要警惕 | 衡量现金流刚性 |
| 债务偿付率 | 月债务还款 / 税后收入 | 消费债越低越好 | 衡量债务压力 |
| 应急金覆盖月数 | 应急金余额 / 月必要开支 | 达到目标月数 | 衡量安全垫 |
| 投资率 | 长期投资 / 税后收入 | 因阶段而异 | 衡量长期增长投入 |

储蓄率与 FIRE 的关系可回到 [FIRE 基础](01-fire-basics.md) 继续推导：储蓄率提升同时提高投资本金、降低未来所需生活费，是双重杠杆。

### 6.2 50/30/20 预算框架

| 桶 | 比例 | 包含 |
| --- | ---: | --- |
| Needs 必需 | 50% | 住房、食物、交通、保险、最低还款 |
| Wants 想要 | 30% | 娱乐、旅行、购物、订阅 |
| Savings 储蓄/还债 | 20% | 应急金、投资、额外还债 |

示例：税后月收入 40,000。

| 桶 | 目标金额 | 实际例子 |
| --- | ---: | --- |
| Needs 50% | 20,000 | 房租 12,000 + 食物 4,000 + 交通 1,500 + 保险 1,500 + 水电 1,000 |
| Wants 30% | 12,000 | 外食 3,000 + 购物 3,000 + 娱乐 2,000 + 旅行基金 4,000 |
| Savings 20% | 8,000 | 应急金 5,000 + 指数基金 3,000 |

适用：刚开始做预算、收入稳定、需要一个简单默认值。不适用：大城市住房占比过高、FIRE 目标激进、正在清高息债。

### 6.3 Zero-based budgeting 零基预算

零基预算的原则是：**每一元收入都有任务**。

```text
收入 - 必需支出 - 目标储蓄 - 投资 - 可选消费 - sinking funds = 0
```

这里的 0 不是花光，而是“未分配金额为 0”。

示例：税后月收入 50,000。

| 类别 | 预算 | 说明 |
| --- | ---: | --- |
| 房租/房贷 | 14,000 | 固定 |
| 食物 | 5,000 | 必需 |
| 水电通信 | 1,500 | 必需 |
| 交通 | 2,000 | 必需 |
| 保险 | 2,000 | 必需 |
| 医疗 | 800 | 必需 |
| 应急金 | 8,000 | 目标储蓄 |
| 长期投资 | 7,000 | 指数基金/养老金等 |
| 年度保险 sinking fund | 1,500 | 年付账单月度化 |
| 旅行 sinking fund | 2,000 | 目标支出 |
| 娱乐/外食 | 3,000 | 可选 |
| 购物 | 2,000 | 可选 |
| 机动 | 1,200 | 小额误差 |
| **合计** | **50,000** | 未分配 = 0 |

执行规则：月初写预算；每笔支出记录到类别；某类超支时从另一类转移预算；月末剩余金额分配到应急金、投资或 sinking funds。

### 6.4 两种框架如何选择

| 你的情况 | 推荐框架 |
| --- | --- |
| 刚开始、讨厌复杂表格 | 50/30/20 |
| 现金流紧张、需要找漏点 | 零基预算 |
| 收入高、目标多 | 零基预算 + 自动化 |
| 双职工家庭 | 零基预算 + 共享月度 review |
| 自由职业 | 零基预算 + 12 个月滚动现金流预测 |
| FIRE 加速阶段 | 零基预算 + 高储蓄率目标 |

## 7. Sinking funds：把不定期大额支出“月度化”

Sinking fund 是为特定未来支出建立的小资金池。

| 目标 | 周期 | 示例金额 | 每月应存 |
| --- | --- | ---: | ---: |
| 年度保险 | 12 个月 | 12,000 | 1,000 |
| 春节/节日礼金 | 12 个月 | 18,000 | 1,500 |
| 旅行 | 10 个月 | 20,000 | 2,000 |
| 换手机/电脑 | 24 个月 | 18,000 | 750 |
| 车辆保养 | 12 个月 | 6,000 | 500 |
| 医疗自付额 | 12 个月 | 12,000 | 1,000 |
| 子女教育材料 | 12 个月 | 9,600 | 800 |
| 房屋维修 | 12 个月 | 24,000 | 2,000 |

公式：

```text
每月应存 = (目标金额 - 当前余额) ÷ 距离付款月数
```

例子：12 月要支付 12,000 的保险，现在是 7 月，当前余额 2,000，距离付款还有 5 个月：

```text
每月应存 = (12,000 - 2,000) ÷ 5 = 2,000
```

| 项目 | 应急金 | Sinking fund |
| --- | --- | --- |
| 事件性质 | 不确定、突发 | 已知或大概率会发生 |
| 示例 | 失业、医疗垫付、紧急维修 | 年保费、旅行、换电脑 |
| 使用规则 | 只在真正紧急时动用 | 到期按计划使用 |
| 补充方式 | 动用后优先补回 | 每月固定累积 |
| 投资属性 | 安全 + 流动 | 看时间期限，可略微分层 |

不要用应急金支付已知的大额消费。否则应急金会被“计划内支出”慢慢掏空。

## 9. 样例月度预算：从收入到投资的完整闭环

假设一个家庭：税后月收入 52,000；当前应急金 90,000；月必要开支 28,000；目标应急金 6 个月 = 168,000；高息债 0；每月目标储蓄率 35%。

| 类别组 | 类别 | 预算 | 必要/可选 | 自动化方式 |
| --- | --- | ---: | --- | --- |
| Income | 税后工资 | 52,000 | 收入 | 发薪日入账 |
| Needs | 房租/房贷 | -14,000 | 必要 | 自动扣款 |
| Needs | 食物 | -5,500 | 必要 | 信用卡全额还款 |
| Needs | 水电通信 | -1,500 | 必要 | 自动扣款 |
| Needs | 交通 | -2,000 | 必要 | 月卡/油费预算 |
| Needs | 保险 | -2,000 | 必要 | 年付 sinking fund |
| Needs | 医疗 | -1,000 | 必要 | 预算内报销后调整 |
| Needs | 子女/赡养 | -2,000 | 必要 | 自动转账 |
| Sinking | 年度保险 | -1,500 | 目标 | 目标账户 |
| Sinking | 旅行基金 | -2,000 | 可选目标 | 目标账户 |
| Savings | 应急金 | -8,000 | 储蓄 | 发薪日自动转账 |
| Investing | 长期指数投资 | -6,000 | 投资 | 自动定投 |
| Wants | 外食娱乐 | -3,000 | 可选 | 类别预算 |
| Wants | 购物订阅 | -2,000 | 可选 | 类别预算 |
| Buffer | 机动 | -1,500 | 缓冲 | 留在 checking |
| **合计** | **未分配** | **0** |  |  |

指标计算：

```text
必要支出 = 14,000 + 5,500 + 1,500 + 2,000 + 2,000 + 1,000 + 2,000 = 28,000
总储蓄/投资 = 应急金 8,000 + 长期投资 6,000 + sinking funds 3,500 = 17,500
储蓄率 = 17,500 ÷ 52,000 = 33.65%
应急金覆盖月数 = 90,000 ÷ 28,000 = 3.21 个月
距离目标 = 168,000 - 90,000 = 78,000
预计补满 = 78,000 ÷ 8,000 = 9.75 个月
```

如果想把储蓄率提高到 40%，需要每月储蓄/投资：

```text
52,000 × 40% = 20,800
缺口 = 20,800 - 17,500 = 3,300
```

| 调整项 | 月度改善 |
| --- | ---: |
| 外食娱乐从 3,000 降到 2,000 | +1,000 |
| 购物订阅从 2,000 降到 1,200 | +800 |
| 旅行基金从 2,000 暂降到 1,000 | +1,000 |
| 机动从 1,500 降到 1,000 | +500 |
| **合计** | **+3,300** |

这样不会动必要支出，也不会影响保险和长期投资，只是临时降低可选消费和旅行目标。

### 10.3 可运行的本地汇总脚本（可选）

如果你愿意用 Python 快速汇总 CSV，可保存为 `cashflow_summary.py` 并在本地运行。这里给的是独立示例，不要求加入仓库。

```python
import csv
from collections import defaultdict

MONTH = "2026-07"
income = 0.0
expenses = 0.0
necessary = 0.0
by_category = defaultdict(float)

with open("transactions.csv", newline="", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for row in reader:
        if row["Month"] != MONTH:
            continue
        amount = float(row["Amount"])
        category = row["Category"]
        if amount > 0 and row["Type"] == "Income":
            income += amount
        elif amount < 0 and row["Type"] == "Expense":
            expenses += -amount
            by_category[category] += -amount
            if row["IsNecessary"].upper() == "TRUE":
                necessary += -amount

savings = income - expenses
savings_rate = savings / income if income else 0

print(f"Month: {MONTH}")
print(f"Income: {income:,.0f}")
print(f"Expenses: {expenses:,.0f}")
print(f"Necessary spending: {necessary:,.0f}")
print(f"Savings: {savings:,.0f}")
print(f"Savings rate: {savings_rate:.2%}")
print("Top categories:")
for category, amount in sorted(by_category.items(), key=lambda x: x[1], reverse=True)[:5]:
    print(f"- {category}: {amount:,.0f}")
```

运行：

```bash
python cashflow_summary.py
```

预期输出：

```text
Month: 2026-07
Income: 52,000
Expenses: 31,920
Necessary spending: 23,420
Savings: 20,080
Savings rate: 38.62%
Top categories:
- Rent: 14,000
- Food: 5,800
- Entertainment: 3,200
- Transport: 2,000
- Insurance: 1,500
```

实际交易数据可能包含隐私，不要上传到不可信服务。

## 11. 动用应急金的规则

应急金可以用，但要有规则。否则它会变成“心情消费账户”。

### 11.1 三问判断法

动用前问三件事：

1. **是否必要？** 不支付会影响住房、健康、工作能力、家庭责任或信用记录吗？
2. **是否紧急？** 是否不能等到下一次发薪或 sinking fund 到位？
3. **是否意外？** 是否不是已经知道会发生的年度账单或计划消费？

三个答案都是“是”，才动用应急金。

| 事件 | 是否动用 | 理由 |
| --- | --- | --- |
| 失业期间支付房租和食物 | 是 | 必要、紧急、意外 |
| 突发医疗自付额 | 是 | 必要且时间敏感 |
| 电脑坏了且影响工作 | 可能 | 若是生产工具，可动用后补回 |
| 年度保险账单 | 否 | 应由 sinking fund 支付 |
| 旅行机票涨价 | 否 | 可选消费 |
| 股票大跌想加仓 | 否 | 应急金不是抄底资金 |
| 双十一购物 | 否 | 可选消费 |

### 11.2 动用后的补回协议

每次动用应急金后，立刻写一条补回计划：

```text
动用日期：2026-07-18
动用原因：紧急牙科治疗
动用金额：6,000
动用后余额：144,000
目标余额：150,000
补回方式：未来 3 个月每月额外转入 2,000
恢复日期：2026-10-31
```

补回优先级：

1. 暂停可选消费增加。
2. 暂停或降低旅行等可选 sinking funds。
3. 暂时降低新增投资比例。
4. 不停止必要保险和最低债务还款。
5. 高息债仍优先于非必要投资。

## 12. 债务、保险与应急金的配合

### 12.1 高息债优先处理

如果你同时有信用卡债和不足 6 个月应急金，不要机械地先攒满 6 个月。

更稳的顺序：

1. 留 1 个月必要开支，避免继续刷卡。
2. 停止新增高息债。
3. 集中还清年化 10%–15% 以上的消费债。
4. 还清后把原还款额转入应急金。
5. 达到 3 个月后，再逐步恢复投资。

示例：

| 项目 | 数值 |
| --- | ---: |
| 信用卡债 | 30,000 |
| 年化利率 | 18% |
| 月必要开支 | 20,000 |
| 当前现金 | 25,000 |
| 每月可结余 | 8,000 |

执行：

1. 保留 20,000 作为 1 个月缓冲。
2. 立即用 5,000 还信用卡。
3. 未来每月 8,000 还债，约 4 个月清掉。
4. 清掉后每月 8,000 转入应急金。
5. 再用约 5 个月补到 3 个月应急金（60,000）。

### 12.2 保险不是投资，但能保护现金流

应急金解决中小型冲击，保险解决低频高损失事件。

| 风险 | 应急金是否足够 | 更合适工具 |
| --- | --- | --- |
| 1–3 个月失业 | 通常足够 | 应急金 |
| 小额维修 | 通常足够 | 应急金 / sinking fund |
| 大额医疗 | 可能不够 | 医疗保险 + 应急金自付额 |
| 家庭主要收入者身故 | 不够 | 定期寿险 |
| 严重意外导致收入中断 | 不够 | 意外/残障保障（视地区） |
| 长期护理 | 不够 | 专门保障或家庭计划 |

原则：保障型保险用于转移灾难性风险；不把理财型保险当作核心投资工具；保险保费应进入必要开支或 sinking fund，避免到期时动用应急金。

## 13. 月度现金流 review：30 分钟流程

每月固定一天，例如发薪后第一个周末，做一次 review。

### 13.1 准备材料（5 分钟）

- 银行/checking 月末余额。
- HYSA/货币基金/money market/T-bills 余额。
- 信用卡账单。
- 本月工资和副业收入。
- 投资账户本月新增投入金额。
- 大额支出列表。

### 13.2 更新指标（10 分钟）

| 指标 | 本月 | 上月 | 变化 | 行动 |
| --- | ---: | ---: | ---: | --- |
| 税后收入 |  |  |  |  |
| 必要支出 |  |  |  |  |
| 总支出 |  |  |  |  |
| 储蓄率 |  |  |  |  |
| 应急金余额 |  |  |  |  |
| 应急金覆盖月数 |  |  |  |  |
| 高息债余额 |  |  |  |  |
| Sinking funds 覆盖率 |  |  |  |  |

### 13.3 找出 3 个异常（10 分钟）

| 异常 | 判断规则 | 处理动作 |
| --- | --- | --- |
| 必要支出上升 | 连续 2 个月高于预算 10% | 更新预算或找原因 |
| 可选消费超支 | 某类超预算 > 20% | 下月降低该类预算或设置限额 |
| 应急金下降 | 非计划动用 | 写补回计划 |
| 信用卡未全额还款 | 出现利息 | 停止新增投资，优先清债 |
| 年度账单临近 | sinking fund 不足 | 增加月供或削减可选消费 |
| 储蓄率下降 | 低于目标 3 个百分点以上 | 找最大支出类别调整 |

### 13.4 设定下月动作（5 分钟）

只选 1–3 个动作，避免过载：

- 下月应急金自动转账从 5,000 提到 6,000。
- 取消 2 个未使用订阅，预计每月 +200。
- 外食预算从 3,000 调到 2,400。
- 把年保费 sinking fund 每月从 1,000 调到 1,500。
- 信用卡账单改为自动全额还款。
- 把 checking 中超过 1 个月账单缓冲的现金转入 HYSA。

### 13.5 月度 review checklist

- [ ] 已记录本月税后收入。
- [ ] 已记录所有账户月末余额。
- [ ] 已确认信用卡是否全额还款。
- [ ] 已更新应急金余额。
- [ ] 已计算应急金覆盖月数。
- [ ] 已计算储蓄率。
- [ ] 已检查 3 个最大支出类别。
- [ ] 已检查所有 sinking funds 是否按进度。
- [ ] 已确认下月自动转账金额。
- [ ] 已写下 1–3 个下月改进行动。

## 14. 常见坑与排查表

| 坑 | 症状 | 后果 | 修正方法 |
| --- | --- | --- | --- |
| 用月收入算应急金 | 目标过大或过小 | 现金配置失真 | 改用月必要开支 |
| 把应急金投股票 | 市场下跌时不敢取 | 危机中被迫亏损卖出 | 只用现金/短期低波动工具 |
| 全部放长期定期/锁定产品 | 急用时取不出 | 现金流断裂 | 至少 1–2 个月放高流动账户 |
| 没有 sinking funds | 年度账单打断投资 | 应急金被计划支出消耗 | 把大额支出月度化 |
| 预算类别太细 | 记账 2 周后放弃 | 数据不可持续 | 先用 10–15 个大类 |
| 只看收益率不看流动性 | 为多 0.5% 锁一年 | 真正急用时不方便 | 应急金优先 liquidity |
| 信用卡只还最低额 | 现金流看似宽松 | 高息滚动 | 建 1 个月缓冲后优先清债 |
| 发薪后先消费 | 月末没钱储蓄 | 储蓄率不稳定 | 发薪日自动转账 |
| 夫妻/伴侣信息不透明 | 一方不知道账户和规则 | 紧急时无法取用 | 共享账户清单和应急流程 |
| 永远不调整目标 | 生子/换房后仍按旧目标 | 缓冲不足 | 重大生活变化后重算 |

## 15. 从今天开始的 7 天实操计划

### Day 1：列账户

- [ ] 列出所有银行账户、货币基金、券商现金、信用卡、贷款。
- [ ] 标注每个账户的余额、流动性、用途。
- [ ] 找出“真正可用于应急”的现金余额。

### Day 2：算必要开支

- [ ] 收集最近 3 个月账单。
- [ ] 分成住房、食物、交通、保险、医疗、债务、家庭责任、可选消费。
- [ ] 删除危机中可暂停的消费。
- [ ] 得到月必要开支。

### Day 3：定目标月数

- [ ] 用打分法评估收入稳定性和家庭责任。
- [ ] 选择 3、6、9 或 12 个月。
- [ ] 计算目标金额、缺口、预计完成月数。

### Day 4：开或指定专用账户

- [ ] 选择 HYSA、货币基金、money market 或其他高流动工具。
- [ ] 确认赎回到账时间和限额。
- [ ] 不绑定日常消费卡。

### Day 5：设置自动转账

- [ ] 发薪日后自动转入固定金额。
- [ ] 设置信用卡自动全额还款。
- [ ] 设置房租、保险等固定账单提醒或自动扣款。

### Day 6：建立 sinking funds

- [ ] 列出未来 12 个月大额支出。
- [ ] 对每个目标计算每月应存。
- [ ] 把它们加入预算。

### Day 7：做第一次月度 review

- [ ] 记录收入、支出、应急金余额。
- [ ] 计算储蓄率和覆盖月数。
- [ ] 写 1–3 个下月动作。

## 16. 完成标准：做到什么程度算达标

基础达标：

- [ ] 知道自己的月必要开支。
- [ ] 有至少 1 个月必要开支的现金缓冲。
- [ ] 没有滚动高息信用卡债。
- [ ] 应急金与日常消费账户分开。
- [ ] 发薪日有自动储蓄动作。

稳健达标：

- [ ] 应急金达到适合自己的目标月数。
- [ ] 应急金至少分成即时流动层和核心缓冲层。
- [ ] 年度账单和可预见大额支出都有 sinking fund。
- [ ] 每月计算储蓄率。
- [ ] 信用卡自动全额还款。

进阶达标：

- [ ] 有 12 个月滚动现金流预测。
- [ ] 能解释 50/30/20 与零基预算的取舍。
- [ ] 能在失业情景下列出 3 个月、6 个月、12 个月行动方案。
- [ ] 重大生活变化后会重算应急金目标。
- [ ] 长期投资计划不会被普通年度账单打断。

## 小结

投资的胜负，很大部分在开始投之前就已经被现金流结构决定：**应急金兜底、清掉高息债、把大额支出月度化、把储蓄率做高、把加薪当成最好的理财**。地基稳了，[指数基金](03-index-investing-guide.md) 和复利才能安心替你长期工作；如果你追求 FIRE，再回到 [FIRE 基础](01-fire-basics.md) 把储蓄率、资产配置和目标年限连起来。
